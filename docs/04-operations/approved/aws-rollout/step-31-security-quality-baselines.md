# 31단계. 보안 스캔 및 품질 관리 기준 확립

## 목적
- 애플리케이션과 인프라 코드 전반에 적용할 보안/품질 점검 항목을 정의한다.
- CI/CD 파이프라인(27~30단계)에 자동화된 스캔 단계를 통합하고 실패 기준을 수립한다.

## 선행조건
- 7단계에서 정의한 보안 및 규제 요구사항.
- 21단계 아티팩트 전략과 30단계 빌드 파이프라인 설계 결과.
- 27단계 CI/CD 요구사항 정의 문서.

## 필요한 입력 자료
- 소스 코드 언어, 프레임워크, 아티팩트 유형 목록.
- 외부 라이선스 의존성, 컴플라이언스 요구사항 목록.
- 인프라 코드(IaC), 컨테이너 이미지, 런타임 환경 정보.

## 상세 절차
1. **스캔 범위 정의**
   - 애플리케이션 코드(SAST), 의존성(SCA), 컨테이너 이미지, 인프라 코드, 런타임 환경을 포함한다.
   - 3단계 버전 전략에 따라 브랜치별 필수 스캔 대상과 빈도를 명시한다.
2. **도구 선정 및 통합 계획 수립**
   - 관리형 서비스(AWS CodeGuru, Amazon Inspector)와 서드파티 도구(Snyk, SonarQube, Trivy 등)를 비교한다.
   - 기존 라이선스, 온프레미스 연동 요구, 6단계 예산을 고려해 선정한다.
3. **품질 게이트 기준 설정**
   - 취약점 심각도 임계값, 코드 커버리지, 정적 분석 규칙 위반 허용 한도를 정의한다.
   - 7단계 보안 정책과 8단계 거버넌스 원칙(태깅, 정책 준수)을 반영한다.
4. **파이프라인 통합 설계**
   - 30단계 빌드 파이프라인 단계별로 스캔 실행 위치(빌드 전/후)를 결정한다.
   - 28~29단계에서 선정한 CI/CD 도구에서의 실행 방법(액션, 플러그인, 컨테이너)을 문서화한다.
5. **결과 보고 및 보안 운영 흐름 정의**
   - 실패 시 자동 차단, 재시도, 예외 승인(34단계) 절차를 정한다.
   - 26단계 모니터링 플랫폼과 연동해 경고/대시보드를 구성한다.
6. **정기 재평가 및 도구 검증 계획 수립**
   - 분기별 룰셋 리뷰, 취약점 DB 업데이트, 도구 성능 검증 절차를 포함한다.
   - 7단계 규제 감사 요구를 충족하도록 증적 저장소를 정의한다.

## 의사결정 포인트
- SaaS 기반 스캐너 사용 여부 및 데이터 거버넌스 이슈.
- 빌드 파이프라인 실패 기준(차단 vs. 경고)과 예외 처리 권한자.

## 체크리스트
- [ ] 스캔 범위 및 도구가 문서화되었다.
- [ ] 품질 게이트 임계값과 예외 프로세스가 승인되었다.
- [ ] CI/CD 파이프라인 통합 계획이 작성되었다.
- [ ] 보고/감사 증적 관리 방안이 정의되었다.
