# 45단계. 빌드 환경 변수 및 시크릿 주입

## 목적
- CI 파이프라인이 빌드/테스트 수행에 필요한 환경 변수와 시크릿을 안전하게 관리하고 주입하도록 구성한다.
- 시크릿 노출을 방지하고 회전·감사 절차를 확립하여 22, 38단계 정책을 준수한다.

## 선행조건
- [22단계 런타임 환경 변수/시크릿 관리 방안 결정](docs/04-operations/approved/aws-rollout/step-22-runtime-secrets-management.md).
- [38단계 비밀 관리 절차 문서화](docs/04-operations/approved/aws-rollout/step-38-secrets-operations.md).
- 43단계 CI 인프라, 44단계 리포지토리 연동 설정 완료.

## 필요한 입력 자료
- 빌드/테스트에 필요한 환경 변수 및 시크릿 목록.
- 시크릿 저장소(Secrets Manager, SSM Parameter Store, GitHub OIDC 등) 접근 정책.
- 시크릿 회전 일정, 감사 로그 요구사항, 암호화 키 정보.

## 상세 절차
1. **시크릿 분류 및 민감도 평가**
   - API 키, 데이터베이스 자격 증명, 서명 키 등 민감 정보를 분류한다.
   - 7단계 보안 요구와 31단계 품질 기준을 고려해 주입 필요 여부를 평가한다.
2. **저장소 및 주입 방식 선택**
   - Secrets Manager, Parameter Store, CI 전용 시크릿 기능 중 하나를 선택한다.
   - AssumeRole 또는 OIDC 기반 임시 자격 증명을 사용하도록 설계한다.
3. **권한 및 암호화 구성**
   - 41단계 IAM 역할에 필요한 읽기 권한과 조건부 MFA, 태깅 규칙을 적용한다.
   - 시크릿을 KMS 키로 암호화하고, 회전 정책을 등록한다.
4. **파이프라인 변수 설정**
   - 파이프라인 정의 파일에서 시크릿을 참조하도록 구성하고 로컬 변수에 저장하지 않는다.
   - 환경별(dev/test/prod)로 변수 세트를 분리하고 네이밍 규칙을 적용한다.
5. **감사 및 회전 자동화 구성**
   - CloudTrail, Secrets Manager 감사 로그를 활성화하고 76단계 감사 검토에 대비한다.
   - 회전 람다 또는 자동화 스크립트를 설정하고 테스트 일정을 잡는다.
6. **검증 및 교육**
   - 빌드 로그에 시크릿이 노출되지 않는지 확인한다.
   - 운영/개발 팀에 시크릿 취급 절차를 교육하고 위반 시 대응 방안을 문서화한다.

## 의사결정 포인트
- 시크릿 저장소를 단일화할지 서비스별로 분리할지 여부.
- 회전 주기 및 승인 절차를 자동화할지 수동으로 운영할지 결정.
- CI 로그 및 아티팩트 보존 기간과 마스킹 수준.

## 체크리스트
- [ ] 모든 빌드 시크릿이 중앙 저장소에 등록되고 태깅되었다.
- [ ] 파이프라인에서 시크릿이 안전하게 주입되고 로그에 노출되지 않는다.
- [ ] 회전 일정 및 책임자가 문서화되었다.
- [ ] 감사 로그 및 경보 설정이 완료되었다.
