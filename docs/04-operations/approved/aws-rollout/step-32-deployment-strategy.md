# 32단계. 배포 전략 정의

## 목적
- 서비스 특성과 가용성 요구사항에 맞는 배포 전략(블루/그린, 카나리, 롤링 등)을 선택한다.
- 배포 전략을 구현하기 위한 기술적 요건과 운영 절차를 정리한다.

## 선행조건
- 20단계에서 선택한 컴퓨팅 서비스 및 29단계 CD 도구 결정.
- 21단계 아티팩트 전략, 30단계 빌드 파이프라인 설계 결과.
- 26단계 모니터링 체계와 31단계 품질 게이트 기준.

## 필요한 입력 자료
- 서비스 SLA, SLO, 가용성/성능 요구사항.
- 트래픽 패턴, 배포 시간대 제약, 릴리스 빈도 계획.
- 인프라 확장성, 네트워크 토폴로지(14~19단계). 

## 상세 절차
1. **요구사항 수집 및 분석**
   - 비즈니스 연속성 요구, 다운타임 허용 범위를 이해한다.
   - 0단계 요구사항 문서와 7단계 보안/규제 조건을 확인한다.
2. **후보 전략 비교**
   - 블루/그린, 카나리, 롤링, 셀프힐링(오토스케일) 등 전략의 장단점을 비교한다.
   - 20단계에서 선택한 서비스(ECS/EKS/Lambda/EC2)에 적용 가능한지를 평가한다.
3. **기술 구현 세부안 작성**
   - 트래픽 전환 방식(Route 53, ALB, 서비스 메쉬), 헬스체크, 오토스케일 정책을 정의한다.
   - IaC 템플릿(4단계)에 배포 전략 파라미터(가중치, 배치 크기)를 반영한다.
4. **관측 및 검증 계획 수립**
   - 26단계 모니터링 지표와 31단계 품질 게이트 결과를 기반으로 성공 기준을 정한다.
   - 카나리/블루그린 시 메트릭 비교와 자동 롤백 트리거를 정의한다.
5. **운영/커뮤니케이션 프로세스 문서화**
   - 배포 일정 공지, 승인(34단계) 및 롤백(33단계) 흐름과의 연계를 기록한다.
   - 배포 체크리스트와 온콜 알림 절차를 작성한다.
6. **파일럿 및 검증**
   - 스테이징 환경에서 파일럿 배포를 수행하고 절차를 검증한다.
   - 문제 발견 시 전략 수정 또는 복합 전략(예: 카나리 후 블루/그린)을 고려한다.

## 의사결정 포인트
- 다운타임 허용 여부에 따른 전략 선택.
- 트래픽 전환 자동화 수준과 책임자 지정.

## 체크리스트
- [ ] 배포 전략 및 성공 기준이 문서화되었다.
- [ ] 트래픽 전환/모니터링/롤백 구성 요소가 정의되었다.
- [ ] 스테이징 파일럿 테스트 결과가 검토되었다.
- [ ] 운영 커뮤니케이션/온콜 절차가 합의되었다.
