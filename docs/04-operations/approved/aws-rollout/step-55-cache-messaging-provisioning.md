# 55단계. 캐시/메시지 서비스 생성

## 목적
- 24단계에서 결정한 캐시 또는 메시지 브로커 서비스를 실제로 생성하고 보안, 모니터링 구성을 적용한다.
- 애플리케이션에서 사용할 엔드포인트, 인증 정보, 운영 절차를 정리한다.

## 선행조건
- [24단계 캐시/메시징 계획](docs/04-operations/approved/aws-rollout/step-24-cache-messaging-planning.md).
- 17단계 네트워크 보안 정책, 37단계 태깅 규칙, 45단계 시크릿 주입 구성.
- 41단계 IAM 역할 및 42단계 핵심 리소스 생성.

## 필요한 입력 자료
- 선택한 서비스의 구성 파라미터(예: Redis cluster mode, SQS DLQ 설정).
- 접근 제어 정책(IAM 정책, 보안 그룹, VPC 엔드포인트).
- 모니터링 요구사항(지표, 알람 임계값, 로그 수집).

## 상세 절차
1. **구성 파라미터 확정**
   - 노드 수, 샤드 수, 메시지 보존 기간, 큐 속성 등 세부 파라미터를 결정한다.
   - 고가용성/백업 옵션을 선택하고 비용 영향을 검토한다.
2. **네트워크 및 보안 구성**
   - 프라이빗 서브넷 배치, 보안 그룹 인바운드 규칙, IAM 액세스 정책을 설정한다.
   - KMS 암호화 여부와 키 관리 방식을 결정한다.
3. **리소스 생성**
   - IaC 또는 AWS CLI/콘솔을 사용해 ElastiCache, SQS/SNS, MSK 등 서비스를 생성한다.
   - 생성 후 엔드포인트, ARN, 리소스 ID를 기록한다.
4. **모니터링 및 로깅 설정**
   - CloudWatch 지표/알람, 로그 스트림, 이벤트 알림(SNS, EventBridge)을 구성한다.
   - 58단계 모니터링 인프라 생성 시 연동할 대상을 정의한다.
5. **성능 및 장애 대응 계획 수립**
   - 캐시 클러스터 백업, 장애 조치 전략, 큐 DLQ/리드라이브 정책을 문서화한다.
   - 77단계 자동 스케일링 설계 시 참조할 기준을 마련한다.
6. **접근 정보 배포**
   - 엔드포인트/자격 증명을 52단계 설정 템플릿 또는 시크릿 저장소에 반영한다.
   - 애플리케이션 코드/인프라에서 사용할 환경 변수를 정의한다.

## 의사결정 포인트
- 단일 노드 vs. 클러스터, 표준 vs. 프로비저닝된 처리량 등 규모 선택.
- 암호화 및 인증 방식(Redis AUTH, IAM, VPC 엔드포인트 정책 등).
- 메시지 재처리 및 장애 시나리오 대응 전략.

## 체크리스트
- [ ] 리소스가 올바른 네트워크/보안 설정으로 생성되었다.
- [ ] 모니터링/알람이 구성되었다.
- [ ] 엔드포인트 및 자격 증명이 문서화되고 안전하게 배포되었다.
- [ ] 백업/장애 대응 전략이 수립되었다.

## 산출물 및 보관 위치
- IaC 스택/모듈 (`infra/cache/`, `infra/messaging/`).
- 운영 가이드 (`docs/runbooks/cache-operations.md`, `docs/runbooks/messaging-operations.md`).
- 모니터링/알람 설정 문서 (`docs/reports/cache-messaging-monitoring.md`).
