# 65단계. 개발 환경 최초 배포 실행

## 목적
- CD 파이프라인을 통해 개발 환경(dev)에 애플리케이션과 인프라를 최초로 배포한다.
- 배포 결과와 모니터링 지표를 분석하여 향후 반복 배포의 기준선을 수립한다.

## 선행조건
- [64단계 CD 드라이 런](docs/04-operations/approved/aws-rollout/step-64-cd-dry-run.md)에서 변경 계획을 검증 완료.
- [60단계 설정 값 검증](docs/04-operations/approved/aws-rollout/step-60-configuration-validation.md) 결과를 적용해 구성 오류를 최소화한다.
- [58단계 모니터링/로깅 인프라](docs/04-operations/approved/aws-rollout/step-58-monitoring-logging-provisioning.md)와 [59단계 로그 보존 정책](docs/04-operations/approved/aws-rollout/step-59-log-collection-retention.md)이 활성화되어 있어야 한다.

## 필요한 입력 자료
- 배포 대상 아티팩트 버전(컨테이너 이미지 태그, 패키지 버전 등).
- 환경별 파라미터 파일, 시크릿 스토어 경로.
- 배포 실행 계획서(드라이 런 결과, 승인 기록).

## 상세 절차
1. **배포 창 및 승인 확인**
   - 34단계 승인 정책에 따라 배포 창을 확보하고 이해관계자의 승인을 확인한다.
   - 장애 대응 인력 및 롤백 계획이 준비되었는지 검토한다.
2. **파이프라인 실행**
   - CD 파이프라인을 수동/자동으로 트리거하고 실행 상태를 모니터링한다.
   - 파이프라인 각 스테이지 로그를 저장하여 실패 시 신속히 대응할 수 있도록 한다.
3. **배포 후 검증**
   - 66단계 헬스체크, 67단계 기능 검증을 위한 사전 체크(엔드포인트 응답, 기본 API 호출)를 수행한다.
   - IaC 리소스 상태와 태그, 접근 정책이 기대값과 일치하는지 확인한다.
4. **문제 발생 시 대응**
   - 실패 지점에 따라 롤백 또는 재시도를 수행한다(33단계 롤백 정책 참조).
   - 장애 보고 템플릿에 근거하여 이슈를 기록한다.
5. **결과 공유 및 기록**
   - 배포 로그, 알람 결과, 주요 지표를 수집하여 이해관계자에게 공유한다.
   - 95단계 배포 기록 문서에 실행 내역과 교훈을 기록한다.

## 의사결정 포인트
- 최초 배포 후 자동 롤백을 허용할지 여부.
- 배포 성공 기준(헬스체크 성공, 기능 테스트 통과 등)을 어떻게 정의할지.
- 문제 발생 시 어떤 조건에서 Change Freeze를 선언할지.

## 체크리스트
- [ ] 파이프라인이 오류 없이 완료되었다.
- [ ] 배포된 리소스와 서비스가 기대 상태로 동작한다.
- [ ] 로그와 지표가 정상적으로 수집되고 알람이 동작한다.
- [ ] 배포 결과가 문서화되고 이해관계자에게 공유되었다.

## 산출물 및 보관 위치
- 배포 실행 로그, CloudWatch 대시보드 스냅샷.
- 배포 결과 보고서(`docs/reports/dev-deployment-YYYYMMDD.md`).
- 문제/이슈 티켓, 롤백 실행 기록.
