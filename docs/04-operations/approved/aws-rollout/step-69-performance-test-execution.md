# 69단계. 성능 테스트 실행

## 목적
- 68단계에서 수립한 계획에 따라 성능 테스트를 수행하고 시스템의 확장성, 안정성을 검증한다.
- 결과를 분석하여 용량 계획과 최적화 과제를 도출한다.

## 선행조건
- [68단계 성능 테스트 계획](docs/04-operations/approved/aws-rollout/step-68-performance-test-planning.md) 승인.
- [65단계 최초 배포](docs/04-operations/approved/aws-rollout/step-65-initial-dev-deployment.md) 완료 및 [67단계 기능 검증](docs/04-operations/approved/aws-rollout/step-67-functional-validation.md) 통과.
- 모니터링/로그 수집이 [58·59단계](docs/04-operations/approved/aws-rollout/step-58-monitoring-logging-provisioning.md) 기준대로 동작해야 한다.

## 필요한 입력 자료
- 테스트 도구 스크립트와 인프라.
- 테스트 데이터 세트, 계정, API 키 등.
- 모니터링 대시보드, 알람 설정.

## 상세 절차
1. **테스트 환경 준비**
   - 로드 제너레이터 인스턴스를 배포하고 네트워크 접근 권한을 확인한다.
   - 테스트 데이터 초기화 및 헬스체크 상태를 점검한다.
2. **사전 점검**
   - 모니터링 대시보드가 실시간으로 지표를 수집하는지 확인한다.
   - 비용 알람(71단계 계획)에 영향을 주지 않도록 임계값을 일시 조정하거나 알림 수신자를 공지한다.
3. **테스트 실행**
   - 계획된 시나리오별로 부하를 단계적으로 증가시키며 실행한다.
   - 각 단계에서 응답 시간, 오류율, 리소스 사용량을 기록한다.
4. **이벤트 및 오류 분석**
   - 오류 발생 시 로그, 트레이스, 메트릭을 수집하여 원인을 파악한다.
   - 병목 지점을 식별하고 임시 조치(스케일 아웃, 캐시 조정 등)를 수행한다.
5. **후처리 및 보고서 작성**
   - 테스트 결과를 그래프, 표 형태로 정리한다.
   - 목표 대비 성과를 분석하고 최적화 과제, 용량 증설 필요 여부를 제안한다.
6. **리소스 정리**
   - 테스트에 사용한 인스턴스, 임시 네트워크 설정을 종료하여 비용을 절감한다.
   - 로그와 결과 파일을 장기 보관 위치에 업로드한다.

## 의사결정 포인트
- 성능 목표 미달 시 즉시 튜닝을 진행할지, 후속 단계로 넘길지.
- 테스트 중 서비스 영향이 감지되면 중단할 임계값.
- 결과 공유 범위(개발팀 vs 경영진)와 보고서 포맷.

## 체크리스트
- [ ] 모든 계획된 시나리오가 실행되었다.
- [ ] 주요 지표가 수집되고 목표 대비 분석이 완료되었다.
- [ ] 최적화 과제와 담당자가 지정되었다.
- [ ] 테스트에 사용한 리소스가 정리되었다.

## 산출물 및 보관 위치
- 성능 테스트 결과 보고서(`docs/reports/performance-test-results.md`).
- 지표 대시보드 스냅샷, 로그 아카이브.
- 최적화 백로그 항목.
