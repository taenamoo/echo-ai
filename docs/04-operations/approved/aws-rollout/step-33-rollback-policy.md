# 33단계. 롤백 정책 문서화

## 목적
- 배포 실패 또는 품질 저하 시 신속하게 이전 안정 상태로 되돌리기 위한 절차를 정의한다.
- 자동/수동 롤백 기준과 책임 체계를 마련해 서비스 중단을 최소화한다.

## 선행조건
- 32단계 배포 전략 결정.
- 30단계 빌드 파이프라인 설계와 31단계 품질 게이트 기준.
- 26단계 모니터링 및 로깅 체계.

## 필요한 입력 자료
- 주요 서비스 구성요소 목록 및 의존성 다이어그램(2단계).
- 데이터베이스/스토리지 백업 정책(23, 25단계)과 동기화 요구사항.
- 비즈니스 영향도 분석 및 SLA/RTO/RPO 기준(39단계 초안).

## 상세 절차
1. **롤백 시나리오 식별**
   - 배포 실패, 성능 저하, 보안 이슈, 데이터 손상 등 가능한 실패 시나리오를 정의한다.
   - 각 시나리오별 감지 트리거(26단계 모니터링 지표, 31단계 스캔 결과)를 연결한다.
2. **복구 전략 정의**
   - 애플리케이션: 이전 버전 아티팩트 재배포, 트래픽 재전환 절차를 명시한다.
   - 데이터베이스/스토리지: 스냅샷 복구, 포인트인타임 리커버리, 레플리카 승격 절차를 포함한다.
3. **자동화 수준 결정**
   - 29단계 CD 도구에서 제공하는 자동 롤백 기능 사용 여부를 결정한다.
   - 수동 승인(34단계)과 조합 시 필요한 승인 플로우를 문서화한다.
4. **운영 Runbook 작성**
   - 롤백 실행 단계별 체크리스트, 책임자, 커뮤니케이션 채널을 정의한다.
   - 롤백 후 후속 조치(원인 분석, 고객 통지, 티켓 기록)를 포함한다.
5. **테스트 및 검증 계획 수립**
   - 정기 롤백 연습(게임데이) 일정을 잡고 성공 기준을 정의한다.
   - 스테이징/프리프로덕션 환경에서 롤백 시나리오를 검증하고 개선 사항을 반영한다.

## 의사결정 포인트
- 자동 롤백 트리거 임계값과 수동 개입 필요 여부.
- 데이터 롤백 시점 선택(전체 복구 vs. 부분 복구) 및 책임 부서.

## 체크리스트
- [ ] 주요 실패 시나리오와 감지 트리거가 정의되었다.
- [ ] 애플리케이션/데이터 롤백 절차가 문서화되었다.
- [ ] 자동화 도구 설정 및 수동 승인 흐름이 합의되었다.
- [ ] 정기 테스트 계획과 기록 보관 방안이 마련되었다.

## 산출물 및 보관 위치
