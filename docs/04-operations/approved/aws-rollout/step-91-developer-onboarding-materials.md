# 91단계. 개발자 온보딩 자료 준비

## 목적
- 신규 개발자가 AWS 개발 환경과 배포 절차를 빠르게 이해하고 기여할 수 있도록 표준 온보딩 패키지를 마련한다.
- 0~65단계에서 구축한 인프라와 CI/CD 흐름을 설명해 지식 격차를 최소화하고 운영 실수를 방지한다.

## 선행조건
- [73단계 문서화 정비](docs/04-operations/approved/aws-rollout/step-73-documentation-refresh.md) 결과가 최신 상태여야 한다.
- [74단계 운영 핸드오프 계획](docs/04-operations/approved/aws-rollout/step-74-operations-handoff-plan.md)에 따른 교육 범위와 일정이 합의되어 있어야 한다.
- IAM 계정·역할 구조(12단계), CI/CD 사용 가이드(43~47단계)가 정리되어 있어야 한다.

## 필요한 입력 자료
- 아키텍처 다이어그램, 네트워크 토폴로지, 운영 Runbook 등 핵심 문서.
- 개발 환경 접근 절차, 필수 도구 설치 가이드, 샘플 구성 파일.
- CI/CD 파이프라인 흐름도, 품질 게이트 기준, 배포 승인 절차.

## 상세 절차
1. **온보딩 범위와 대상 정의**
   - 신규 입사자 역할별(백엔드/프론트엔드/DevOps) 필수 학습 항목을 도출한다.
   - 21~30단계 결과물을 검토해 언어나 서비스별로 필요한 배경 지식을 분류한다.
2. **문서 및 자료 수집**
   - 73단계에서 최신화한 문서, 58단계 관측 시스템 가이드, 71단계 비용 알람 운영 지침을 모은다.
   - 자주 묻는 질문(FAQ)과 트러블슈팅 로그를 정리해 참고 자료로 추가한다.
3. **온보딩 패키지 제작**
   - 계정 신청→로컬 환경 설정→CI 빌드→CD 배포 순으로 따라할 수 있는 가이드를 작성한다.
   - 체크리스트, 퀵스타트 영상/스크린샷, 실습 과제를 포함해 이해도를 높인다.
4. **검토 및 승인**
   - 기술 리더, 운영팀과 자료를 리뷰해 정확성을 확인하고 34단계 승인 절차에 따라 배포를 승인받는다.
   - 보안 민감 정보는 22·38단계 정책에 맞춰 마스킹하거나 접근 권한을 제한한다.
5. **온보딩 세션 준비**
   - 74단계 일정에 맞춰 실습 환경을 점검하고 테스트 계정/리소스를 마련한다.
   - 세션 후 설문지를 배포해 개선 의견을 수집하고 99단계 피드백 루프에 반영한다.

## 의사결정 포인트
- 온보딩을 문서 기반 비동기 방식으로 진행할지, 라이브 세션을 병행할지 여부.
- 온보딩 패키지를 사내 위키/학습 플랫폼 어디에 게시할지 결정.
- 실습 환경을 전용 샌드박스 계정으로 제공할지, 공유 개발 계정을 사용할지 선택.

## 체크리스트
- [ ] 온보딩 대상과 역할별 학습 항목이 정의되었다.
- [ ] 최신 문서와 Runbook이 패키지에 포함되었다.
- [ ] 실습 흐름이 계정 신청부터 배포까지 단계별로 정리되었다.
- [ ] 검토 및 승인 절차를 거쳐 자료가 공식 채널에 게시되었다.

## 산출물 및 보관 위치

- 실습 체크리스트 및 설문 결과 (`docs/onboarding/checklists/` 및 `docs/onboarding/feedback/`).
- 교육 세션 녹화/자료 (`knowledge-base/training/developer-onboarding/`).
