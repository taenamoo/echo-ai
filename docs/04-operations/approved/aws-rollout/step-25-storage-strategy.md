# 25단계. 스토리지 전략 수립

## 목적
- 객체, 블록, 파일 스토리지 요구사항을 분석하여 S3, EFS, EBS 등 적합한 서비스를 선택한다.
- 데이터 보존, 암호화, 라이프사이클 관리 정책을 정의한다.

## 선행조건
- 0단계 요구사항, 2단계 아키텍처, 7단계 보안·규제 요구사항.
- 6단계 비용 계획, 20단계 컴퓨팅 서비스 결과.
- 21단계 아티팩트 전략, 23~24단계 데이터 계층 설계.

## 필요한 입력 자료
- 데이터 유형(정적 자산, 로그, 백업, 사용자 업로드 등)과 접근 패턴.
- 보존 기간, 삭제 정책, 규제 준수 요구사항.
- 멀티 AZ/리전 복제, 전송 암호화 필요 여부.

## 상세 절차
1. **데이터 분류 및 사용 패턴 분석**
   - 정적 자산, 트랜잭션 데이터, 백업, 로그/메트릭 데이터를 분류한다.
   - 접근 빈도, 지연 요구, 용량 성장률을 추정한다.
2. **스토리지 서비스 매핑**
   - 객체: S3(Standard/IA/Glacier), 버전 관리 여부.
   - 블록: EBS 볼륨 유형(gp3, io2 등), 스냅샷 전략.
   - 파일: EFS, FSx, 온프레미스 연계 필요 여부.
3. **보존 및 라이프사이클 정책 수립**
   - S3 수명 주기 규칙, Object Lock, Glacier 보관 정책을 정의한다.
   - EBS/EFS 백업 스케줄과 삭제 정책을 문서화한다.
4. **보안·컴플라이언스 설계**
   - KMS 암호화 키, 서버사이드 암호화, 접근 로그(S3 Server Access Logging, CloudTrail)를 설정한다.
   - VPC 엔드포인트, 버킷 정책, IAM 역할을 정의한다.
5. **데이터 이동 및 통합 방안**
   - 온프레미스 마이그레이션 시 Snowball/DataSync 사용 여부를 결정한다.
   - 27~30단계 파이프라인에서 업로드/정리 작업을 자동화한다.
6. **비용 최적화 전략**
   - S3 인텔리전트 티어링, Glacier Deep Archive, EBS Snapshots Archive 활용 여부를 검토한다.
   - 태깅(8단계) 및 비용 알람(36단계)과 연계한다.

## 의사결정 포인트
- 온프레미스 연동(Direct Connect, Storage Gateway) 필요 여부.
- 규제 데이터 격리를 위한 별도 계정/버킷 분리 여부.

## 체크리스트
- [ ] 데이터 유형별 스토리지 서비스가 결정되었다.
- [ ] 보존·라이프사이클·암호화 정책이 문서화되었다.
- [ ] 마이그레이션 및 자동화 절차가 준비되었다.
- [ ] 비용 최적화 및 모니터링 전략이 수립되었다.
