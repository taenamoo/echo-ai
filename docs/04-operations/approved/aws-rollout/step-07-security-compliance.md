# 7단계. 보안 및 규제 요구사항 분석

## 목적
- 서비스가 준수해야 할 보안 및 규제 요구사항을 파악하여 AWS 설계와 운영 정책에 반영한다.
- 감사 추적과 인증 획득에 필요한 증적 수집 계획을 수립한다.

## 선행조건
- 0단계 요구사항 추적 매트릭스.
- 6단계 예산 계획(보안 도구 및 인증 비용 반영 필요).

## 필요한 입력 자료
- 기업 보안 정책, 개인정보보호 지침, 산업 규제(예: GDPR, HIPAA 등).
- AWS 보안 모범 사례 문서(AWS Well-Architected Framework - Security Pillar).
- 기존 감사 보고서, 취약점 진단 결과.

## 상세 절차
1. **규제 및 표준 식별**
   - 비즈니스 영역에 적용되는 법규, 인증(ISO 27001, ISMS 등), 고객 계약 요구사항을 정리한다.
   - 데이터 분류 기준(개인정보, 민감정보 등)을 정의한다.
2. **보안 통제 매핑**
   - NIST CSF, CIS Benchmarks 등 참고 프레임워크와 매핑하여 필요한 통제를 도출한다.
   - AWS 서비스별 보안 기능(CloudTrail, GuardDuty, Config 등) 적용 계획을 세운다.
3. **위험 평가 수행**
   - 자산, 위협, 취약점을 식별하고 위험 우선순위를 산정한다.
   - 필요한 경우 Threat Modeling 세션을 진행한다.
4. **보안 가드레일 설계**
   - 접근 제어, 네트워크 보안, 데이터 암호화, 로깅/모니터링 정책을 정의한다.
   - IaC 코드와 CI/CD 파이프라인에 적용할 보안 검사 도구(TFSec, Checkov 등)를 결정한다.
5. **규제 대응 계획 수립**
   - 필수 감사 및 인증 획득 일정, 책임자, 필요 서류를 정리한다.
   - 개인정보 처리방침, 데이터 보관/파기 정책을 업데이트한다.
6. **승인 및 공유**
   - 보안/법무/컴플라이언스 팀과 리뷰 후 승인을 받는다.
   - 이후 단계에서 참조할 수 있도록 중앙 저장소에 게시한다.

## 의사결정 포인트
- 추가적인 보안 솔루션 도입 필요 여부(SIEM, WAF, CSPM 등).
- 외부 감사 기관 선정 및 감사 일정.

## 체크리스트
- [ ] 적용해야 할 규제와 표준 목록이 정리되었다.
- [ ] 통제 항목과 AWS 서비스 매핑이 문서화되었다.
- [ ] 위험 평가 및 가드레일 설계가 완료되었다.
- [ ] 보안/법무/컴플라이언스 승인 기록이 확보되었다.

## 산출물 및 보관 위치
- 보안 및 규제 요구사항 분석 보고서 (예: `docs/security/security-requirements.md`).
- 위험 평가 결과 및 가드레일 정책 문서.
- 감사 준비 체크리스트.
