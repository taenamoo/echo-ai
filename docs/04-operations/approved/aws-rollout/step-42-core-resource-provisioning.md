# 42단계. S3/ECR 등 핵심 리소스 생성

## 목적
- 21단계 아티팩트 전략과 25단계 스토리지 계획에 따라 아티팩트 저장소(S3, ECR 등) 및 필수 스토리지 리소스를 실제로 생성한다.
- 생성된 리소스에 암호화, 버전 관리, 태깅, 접근 제어 정책을 적용해 운영 표준을 확립한다.

## 선행조건
- [21단계 이미지 및 아티팩트 전략 수립](docs/04-operations/approved/aws-rollout/step-21-artifact-strategy.md).
- [25단계 스토리지 전략 수립](docs/04-operations/approved/aws-rollout/step-25-storage-strategy.md).
- 36단계 비용 모니터링 초안, 37단계 태깅 정책, 22/38단계 시크릿 운영 가이드.

## 필요한 입력 자료
- 생성 대상 리소스 목록(예: S3 버킷, ECR 리포지토리, CodeArtifact 등).
- 암호화 키(KMS), 라이프사이클 정책, 접근 정책 JSON.
- 네트워크/VPC 엔드포인트 요구사항(PrivateLink, VPC 엔드포인트).

## 상세 절차
1. **리소스 명명 및 계정/리전 배치 확정**
   - 5단계 환경 정의에 따라 리전/계정 분리를 확인한다.
   - 8단계 명명 규칙과 37단계 태깅 카탈로그를 적용한다.
2. **보안 및 암호화 설정 준비**
   - S3 버킷에 기본 암호화, 버전 관리, MFA Delete 여부를 설정한다.
   - ECR 리포지토리에 이미지 스캔, 암호화, 수명 주기 규칙을 설정한다.
3. **리소스 생성 실행**
   - IaC 템플릿, AWS CLI, 콘솔 중 선택한 방식으로 리소스를 생성한다.
   - 생성 시 태그, 접근 정책, 퍼블릭 접근 차단 옵션을 즉시 적용한다.
4. **접근 제어 및 네트워크 구성**
   - 41단계에서 만든 IAM 역할에 필요한 권한을 부여한다.
   - VPC 엔드포인트 또는 프라이빗 링크를 설정하여 내부 전용 접근을 보장한다.
5. **비용 및 모니터링 연계**
   - Cost Explorer 태그, AWS Budgets 리소스 그룹에 등록한다.
   - CloudWatch 지표 및 이벤트 알람을 구성하고 26단계 모니터링과 연동한다.
6. **검증 및 문서화**
   - 리소스 접근 테스트(S3 업로드/다운로드, ECR push/pull)를 수행한다.
   - 생성된 리소스와 정책을 문서화하고 형상 관리 저장소에 기록한다.

## 의사결정 포인트
- 리소스별 계정/리전 분산 여부와 크로스계정 접근 방식.
- ECR 이미지 스캔/서명 도구 선택 여부(Inspector, Notary 등).
- S3 버킷용 버전 관리, Glacier 전환 정책 활성화 기준.

## 체크리스트
- [ ] 모든 리소스가 태깅 규칙과 명명 규칙을 준수한다.
- [ ] 암호화 및 퍼블릭 접근 차단 설정이 적용되었다.
- [ ] 접근 테스트(읽기/쓰기)가 성공했다.
- [ ] 비용 모니터링 및 알람 대시보드에 리소스를 등록했다.

## 산출물 및 보관 위치
- 리소스 생성 IaC 템플릿 (`infra/storage/` 혹은 `infra/artifacts/`).
- 접근 정책 및 테스트 로그 (`docs/reports/storage-access-validation.md`).
- 모니터링/비용 대시보드 구성 문서 (`docs/runbooks/cost-monitoring.md`).
