# 93단계. 운영 자동화 스크립트 정리

## 목적
- 반복적인 운영 작업을 자동화하여 배포 이후 운영 효율과 일관성을 확보한다.
- 수동 작업에 의한 오류를 줄이고 94단계 변경 관리 및 95단계 릴리즈 기록과 연계된 자동 보고를 가능하게 한다.

## 선행조건
- [51단계 구성 관리 계획](docs/04-operations/approved/aws-rollout/step-51-configuration-management-plan.md)이 수립되어 있어야 한다.
- [61단계 배포 스크립트 준비](docs/04-operations/approved/aws-rollout/step-61-deployment-script-prep.md)와 [83단계 운영 대시보드](docs/04-operations/approved/aws-rollout/step-83-operations-dashboard.md)가 정상 동작해야 한다.
- 자동화 대상 작업의 Runbook(74단계)과 접근 권한(41단계)이 문서화되어 있어야 한다.

## 필요한 입력 자료
- 현재 수동으로 수행 중인 운영 절차 목록과 작업 빈도, 소요 시간 데이터.
- AWS CLI/SDK, Terraform, Lambda 등 사용할 자동화 도구 및 권한 정보.
- 로그 및 감사 요구사항(76단계)과 보안 정책(81단계).

## 상세 절차
1. **자동화 후보 식별 및 우선순위 설정**
   - 배포 후 반복 실행되는 작업(로그 수집, 헬스체크, 비용 보고 등)을 목록화한다.
   - 영향도와 구현 난이도에 따라 우선순위를 정하고 97단계 정기 회의 안건으로 공유한다.
2. **설계 및 보안 검토**
   - 각 작업에 필요한 입력, 출력, 실패 시 롤백 전략을 정의한다.
   - 권한 최소화 원칙에 따라 IAM 역할을 설계하고 12·38단계 정책과 충돌이 없는지 확인한다.
3. **스크립트 개발 및 테스트**
   - IaC 레포지토리 또는 운영 자동화 레포지토리에 코드를 작성한다.
   - 샌드박스 환경에서 테스트하고 69단계 성능 테스트와 충돌이 없는지 확인한다.
4. **배포 및 스케줄링**
   - AWS Lambda, Step Functions, EventBridge, GitHub Actions 등 실행 환경을 선택해 배포한다.
   - 실행 로그가 58·59단계 모니터링/로그 보존 정책에 따라 수집되는지 검증한다.
5. **문서화 및 교육**
   - 스크립트 사용법, 입력 파라미터, 장애 처리 방법을 Runbook에 추가한다.
   - 91단계 온보딩 자료에 자동화 사용 가이드를 포함하고 담당자에게 교육한다.

## 의사결정 포인트
- 자동화 도구 선택(AWS 네이티브 vs. 외부 파이프라인)과 실행 주기.
- 운영팀/개발팀 중 어느 조직이 소유하고 유지보수할지 결정.
- 실패 시 알림 채널(79단계 Incident 연동)을 어디로 보낼지 지정.

## 체크리스트
- [ ] 자동화 대상과 우선순위가 문서화되었다.
- [ ] 보안 및 권한 검토를 완료하고 필요한 IAM 역할이 생성되었다.
- [ ] 스크립트가 테스트 환경에서 검증되고 실행 로그가 수집된다.
- [ ] 운영 문서와 온보딩 가이드가 업데이트되었다.

## 산출물 및 보관 위치
- 자동화 스크립트 리포지토리(`repos/ops-automation/`) 및 IaC 모듈.
- 실행 로그 및 모니터링 대시보드(`cloudwatch-logs/ops-automation/`, `dashboards/automation.json`).
- 유지보수 Runbook (`runbooks/automation/automation-tasks.md`).
