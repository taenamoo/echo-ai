# 54단계. 데이터베이스 초기화 및 마이그레이션

## 목적
- 새로 생성된 데이터베이스에 스키마, 기본 데이터, 권한을 초기화하고 마이그레이션 도구를 통해 자동화된 배포 절차를 확립한다.
- 향후 배포 시 스키마 변경이 CI/CD 파이프라인에 통합되도록 한다.

## 선행조건
- [53단계 데이터베이스 프로비저닝](docs/04-operations/approved/aws-rollout/step-53-database-provisioning.md).
- [27단계 CI/CD 요구사항](docs/04-operations/approved/aws-rollout/step-27-cicd-requirements.md) 및 [30단계 빌드 파이프라인 설계](docs/04-operations/approved/aws-rollout/step-30-artifact-build-pipeline.md).
- 31단계 품질 기준과 33단계 롤백 정책.

## 필요한 입력 자료
- 스키마 정의서(DDL), 마이그레이션 스크립트 또는 도구(Flyway, Liquibase 등).
- 기본 데이터 시드 목록, 권한 매핑 표.
- 롤백 스크립트 또는 스냅샷 전략.

## 상세 절차
1. **마이그레이션 도구 구성**
   - 버전 관리 저장소 구조(`db/migrations/<version>`)와 네이밍 규칙을 정의한다.
   - 도구 설정 파일을 52단계 설정 템플릿 또는 51단계 구성 관리에 통합한다.
2. **접속 자격 증명 준비**
   - 45단계 시크릿 주입 방식을 사용하여 마이그레이션 도구가 DB에 안전하게 연결되도록 구성한다.
   - 권한 최소화를 위해 마이그레이션 전용 계정을 생성한다.
3. **초기 스키마 및 데이터 적용**
   - 개발/테스트 환경에서 초기 스키마와 시드 데이터를 적용한다.
   - 트랜잭션 사용 여부, 잠금 영향 등을 검토한다.
4. **검증 테스트 작성**
   - 마이그레이션 후 수행할 검증 쿼리/테스트 케이스를 정의한다.
   - 46단계 테스트 스위트에 DB 검증 단계를 통합한다.
5. **롤백 전략 수립**
   - 마이그레이션 실패 시 사용할 다운그레이드 스크립트 또는 스냅샷 복구 절차를 문서화한다.
   - 33단계 롤백 정책과 일치하도록 승인/알림 흐름을 구성한다.
6. **CI/CD 통합**
   - 63단계 CI 리허설에서 마이그레이션 단계가 자동 실행되는지 확인한다.
   - 64단계 CD 드라이런에서 변경 사항이 안전하게 적용되는지 검토한다.

## 의사결정 포인트
- 데이터베이스 마이그레이션 도구 선택 및 호환성.
- 다운타임 허용 여부에 따른 롤링/블루그린 마이그레이션 전략.
- 시드 데이터 관리 방식(코드 내 포함 vs. 외부 스토리지).

## 체크리스트
- [ ] 마이그레이션 스크립트가 버전 관리되고 검토되었다.
- [ ] 테스트 환경에서 초기화/마이그레이션이 성공적으로 완료되었다.
- [ ] 롤백 절차가 실행 가능한 상태로 문서화되었다.
- [ ] CI/CD 파이프라인 통합이 완료되었다.

## 산출물 및 보관 위치
- 마이그레이션 스크립트 (`db/migrations/`).
- 마이그레이션 실행 로그 및 보고서 (`docs/reports/db-migration.md`).
- 롤백 전략 문서 (`docs/runbooks/db-rollback.md`).
