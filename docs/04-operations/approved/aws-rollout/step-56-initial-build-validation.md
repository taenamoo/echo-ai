# 56단계. 애플리케이션 빌드 및 이미지 생성 테스트

## 목적
- CI 환경과 동일한 조건에서 애플리케이션 빌드/이미지 생성 과정을 검증하여 배포 전 오류를 사전에 제거한다.
- 30단계 빌드 파이프라인 정의와 일치하는지 확인하고 빌드 아티팩트 저장소 업로드(48단계) 전 품질을 확보한다.

## 선행조건
- [30단계 아티팩트 빌드 파이프라인 설계](docs/04-operations/approved/aws-rollout/step-30-artifact-build-pipeline.md).
- [41단계 IAM 프로비저닝](docs/04-operations/approved/aws-rollout/step-41-iam-provisioning.md) 및 [42단계 핵심 리소스 생성](docs/04-operations/approved/aws-rollout/step-42-core-resource-provisioning.md).
- 45단계 빌드 시크릿 주입, 46단계 테스트 통합 계획.

## 필요한 입력 자료
- 빌드 스크립트/파이프라인 정의 파일(CI YAML, Makefile 등).
- Dockerfile 또는 패키징 스크립트.
- 테스트 데이터, 환경 변수, 시크릿 참조.

## 상세 절차
1. **빌드 환경 재현**
   - 로컬 또는 임시 CI 환경에서 빌드 컨테이너 이미지/러너 구성을 43단계 CI 인프라와 동일하게 맞춘다.
   - 필요한 환경 변수와 시크릿을 52단계 설정 템플릿을 통해 주입한다.
2. **초기 빌드 실행**
   - 지정된 빌드 명령을 실행하여 아티팩트를 생성한다.
   - 빌드 로그를 수집하고 경고/오류를 분석한다.
3. **이미지/아티팩트 검사**
   - Docker 이미지 메타데이터, 레이어 크기, 취약 라이브러리를 확인한다.
   - 패키지 무결성(SHA, 서명)을 계산한다.
4. **테스트 스위트 연계 확인**
   - 46단계에서 통합한 테스트가 자동으로 실행되는지 확인한다.
   - 실패 테스트는 이슈로 등록하고 재실행한다.
5. **빌드 시간 및 자원 사용량 측정**
   - 빌드/테스트에 소요되는 시간과 리소스를 기록하여 71단계 비용 모니터링에 반영한다.
6. **결과 공유 및 문서화**
   - 빌드 결과물, 로그, 문제 해결 내용을 정리하여 팀에 공유한다.
   - 빌드 파이프라인 정의에 필요한 수정 사항을 반영한다.

## 의사결정 포인트
- 로컬 빌드 vs. 클라우드 CI 샌드박스 사용 여부.
- 캐시(예: Docker layer cache) 활용 전략과 저장 위치.
- 빌드 실패 시 자동 재시도/알림 정책.

## 체크리스트
- [ ] 빌드 및 이미지 생성이 오류 없이 완료되었다.
- [ ] 테스트 스위트가 자동 실행되고 결과가 저장되었다.
- [ ] 아티팩트 무결성/메타데이터가 검증되었다.
- [ ] 개선 사항이 파이프라인 정의에 반영되었다.

## 산출물 및 보관 위치
- 빌드 로그 및 보고서 (`docs/reports/build-smoketest.md`).
- 업데이트된 빌드 스크립트 (`scripts/build/`, `.github/workflows/` 등).
- 아티팩트 메타데이터 기록 (`docs/runbooks/artifact-metadata.md`).
