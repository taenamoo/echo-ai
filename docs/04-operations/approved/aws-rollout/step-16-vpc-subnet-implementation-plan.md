# 16단계. VPC 및 서브넷 생성 계획서 작성

## 목적
- 확정된 네트워크 설계를 기반으로 VPC와 서브넷, 관련 리소스 생성 절차를 상세히 계획한다.
- IaC 또는 수동 프로비저닝 수행 시 필요한 단계와 책임을 명확히 한다.

## 선행조건
- 14단계 네트워크 설계 초안 및 15단계 CIDR 계획.
- 4단계 IaC 도입 여부 결정 결과.
- 11단계 계정 구조 및 12단계 IAM 역할 설계.

## 필요한 입력 자료
- 계정/환경별 리전, AZ, CIDR 배정 표.
- 라우팅, 게이트웨이, 보안 요구사항 상세 목록.
- IaC 모듈 구조 또는 수동 프로비저닝 체크리스트.

## 상세 절차
1. **리소스 매트릭스 작성**
   - VPC, 서브넷, 라우팅 테이블, 게이트웨이, 보안 그룹 등 생성해야 할 리소스 목록을 작성한다.
   - 각 리소스에 대한 계정/환경/리전 정보를 포함한다.
2. **IaC 설계 또는 스크립트 정의**
   - Terraform/CDK 등 IaC 사용 시 모듈 구조, 변수, 상태 백엔드 구성을 상세히 작성한다.
   - 수동 프로비저닝 시 AWS CLI/콘솔 단계별 절차를 정리한다.
3. **종속성 및 순서 계획**
   - 리소스 생성 순서를 정의하고, 선행 조건(예: VPC 생성 후 서브넷)을 명시한다.
   - 네트워크 공유(Account Factory, Resource Access Manager) 사용 여부를 결정한다.
4. **검증 및 테스트 전략 수립**
   - 생성 후 점검할 항목(라우팅, 보안 그룹, CIDR 적용)을 체크리스트화한다.
   - IaC의 경우 `plan`/`apply` 검증 절차와 승인 플로우를 정의한다.
5. **롤백 및 변경 관리 계획**
   - 실패 시 롤백 방법 및 변경 기록 작성 절차를 명시한다.
   - 다중 환경 배포 시 변경 창(Window)과 승인 절차를 정한다.
6. **운영 이관 문서 작성**
   - 운영팀에서 유지보수할 수 있도록 연락처, 장애 대응 방법, 로그 수집 위치를 정리한다.
7. **승인 획득**
   - 네트워크, 보안, 인프라 책임자의 검토와 승인을 받은 후 배포 일정을 확정한다.

## 의사결정 포인트
- 네트워크 공유(VPC 공유, Transit Gateway 연결) 여부.
- IaC 파이프라인 자동화 수준 및 승인 흐름.

## 체크리스트
- [ ] 리소스 매트릭스와 생성 순서가 정의되었다.
- [ ] IaC 코드 또는 수동 절차 문서가 작성되었다.
- [ ] 검증/롤백/운영 계획이 포함되었다.
- [ ] 관련 이해관계자 승인 및 일정이 확정되었다.
