# 64단계. CD 파이프라인 드라이 런

## 목적
- 실제 리소스 변경 없이 CD 파이프라인이 생성하는 변경 사항을 검증해 안전성을 확보한다.
- 61단계 배포 스크립트와 비교하여 누락된 작업이나 파라미터 오류를 조기에 발견한다.

## 선행조건
- [61단계 배포 스크립트 준비](docs/aws-deployment/step-61-deployment-script-prep.md) 완료.
- [49단계 CD 도구 부트스트랩](docs/aws-deployment/step-49-cd-tool-bootstrap.md) 후 파이프라인 정의가 저장소에 반영되어 있어야 한다.
- [63단계 CI 리허설](docs/aws-deployment/step-63-ci-rehearsal.md)에서 산출된 아티팩트가 이용 가능해야 한다.

## 필요한 입력 자료
- CD 파이프라인 템플릿(YAML/JSON) 및 실행 파라미터.
- IaC 템플릿과 상태 파일(Terraform state, CloudFormation stack 등).
- 드라이 런 결과 비교용 기준 자료(이전 실행 로그, 예상 변경 리스트).

## 상세 절차
1. **드라이 런 모드 설정**
   - Terraform: `terraform plan`, CloudFormation: Change Set, Argo CD: `--dry-run`, GitHub Actions: `no-op` 플래그 등 사용 가능한 옵션을 확인한다.
   - 파이프라인에 드라이 런 전용 스테이지를 추가하거나 수동 실행 방법을 정리한다.
2. **파라미터 검증**
   - 52단계 설정 파일, 55단계 서비스 엔드포인트 등 환경별 파라미터가 올바른지 확인한다.
   - 누락된 변수나 권한 오류가 발생하는지 사전 점검한다.
3. **실행 및 결과 수집**
   - 드라이 런을 실행하고 계획된 리소스 추가/변경/삭제 목록을 수집한다.
   - 로그를 저장소 또는 S3에 업로드해 추적 가능성을 확보한다.
4. **61단계 스크립트와 비교 검토**
   - 수동/스크립트 실행 결과와 CD 도구의 변경 계획을 비교하여 차이를 확인한다.
   - 누락된 태그, 파라미터, 의존 리소스가 없는지 점검한다.
5. **승인 및 차이 해결**
   - 34단계 승인 프로세스에 따라 드라이 런 결과를 리뷰한다.
   - 필요한 수정 사항을 IaC 템플릿, 파이프라인 정의에 반영하고 재실행한다.

## 의사결정 포인트
- 드라이 런을 자동화된 파이프라인 스테이지로 유지할지, 수동 승인 단계로 둘지 결정.
- 계획 출력물을 어디에 저장/공유할지(Artefact, S3, PR 코멘트 등).
- 변경 차이가 있을 때 어느 수준까지 자동으로 차단할지 정책 설정.

## 체크리스트
- [ ] 드라이 런이 오류 없이 완료되고 변경 목록을 확보했다.
- [ ] 예상과 다른 변경 사항을 검토하고 조치했다.
- [ ] 결과가 승인 프로세스를 통과했다.
- [ ] 로그와 계획 파일이 보존 위치에 저장되었다.

## 산출물 및 보관 위치
- 드라이 런 계획 파일(`plan/dev-YYYYMMDD.txt`).
- 검토/승인 기록(PR 코멘트, Change Set 승인 이력).
- 파이프라인 정의 업데이트 커밋.
