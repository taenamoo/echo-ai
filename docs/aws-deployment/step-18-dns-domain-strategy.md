# 18단계. DNS 및 도메인 전략 수립

## 목적
- 애플리케이션과 인프라에 필요한 도메인 구조, DNS 관리, 인증서를 계획하여 안정적인 접근을 보장한다.
- 멀티 환경, 멀티 계정에서도 일관된 네이밍과 보안 정책을 적용한다.

## 선행조건
- 6단계 비용 계획 및 8단계 거버넌스 원칙.
- 11단계 멀티 계정 전략과 14단계 네트워크 설계.
- 17단계 네트워크 보안 정책.

## 필요한 입력 자료
- 기존 도메인 자산 목록과 소유권 정보.
- 애플리케이션별 서브도메인 요구사항 및 인증서 필요 서비스.
- 외부 DNS 제공업체 또는 온프레미스 DNS와의 통합 요구.

## 상세 절차
1. **도메인 및 네이밍 표준 정의**
   - 루트 도메인, 환경별 서브도메인 구조(예: `dev.example.com`)를 설계한다.
   - 태깅 및 명명 규칙(8단계)과 일치하도록 문서화한다.
2. **DNS 호스팅 옵션 결정**
   - Route 53, 외부 DNS, 하이브리드 구성을 비교하여 선택한다.
   - 멀티 계정에서 호스팅 존을 공유할지 여부와 접근 제어 방안을 정의한다.
3. **인증서 관리 전략**
   - ACM(공용/사설) 사용 여부를 결정하고 인증서 발급/갱신 절차를 수립한다.
   - 와일드카드, 멀티도메인 인증서 필요성을 검토한다.
4. **레코드 관리 정책**
   - 필수 레코드(A, CNAME, ALIAS, TXT, MX 등) 템플릿을 작성한다.
   - 변경 관리, TTL 정책, 롤백 절차를 정의한다.
5. **보안 및 가용성 고려**
   - DNSSEC, DDoS 보호(Route 53 Resolver DNS Firewall, AWS Shield) 필요성을 평가한다.
   - 헬스체크 및 트래픽 라우팅(지리/가중치 기반) 전략을 설정한다.
6. **운영 및 모니터링 계획**
   - 변경 요청 프로세스, 감사 로그 관리, 경보 설정을 문서화한다.
   - 비용 모니터링 및 최적화 방안을 정의한다.
7. **승인 및 커뮤니케이션**
   - 전략 문서를 이해관계자에게 공유하고 승인받는다.
   - 도메인 등록/이전 일정을 수립하고 관련 팀에 안내한다.

## 의사결정 포인트
- Route 53 단독 사용 vs. 기존 DNS와의 하이브리드 구성.
- 인증서 자동화(AWS Certificate Manager, 외부 ACME) 도입 여부.

## 체크리스트
- [ ] 도메인 및 서브도메인 구조가 정의되었다.
- [ ] DNS 호스팅 및 접근 제어 전략이 확정되었다.
- [ ] 인증서 발급/갱신 절차가 문서화되었다.
- [ ] 운영/모니터링/비용 관리 계획이 승인되었다.
