# 98단계. 향후 기능 확장 대비 로드맵 업데이트

## 목적
- 배포 이후 확보된 피드백과 성능 데이터를 기반으로 향후 기능 확장 및 인프라 개선 로드맵을 최신화한다.
- 9단계 초기 로드맵을 검증하고, 리소스/예산 계획을 조정해 지속적인 개선 방향을 제시한다.

## 선행조건
- [97단계 정기 리뷰 회의](docs/aws-deployment/step-97-regular-review-meetings.md)에서 도출된 결정 사항과 개선 과제가 정리되어 있어야 한다.
- [86단계 비용 최적화 검토](docs/aws-deployment/step-86-cost-optimization-review.md), [88단계 용량 계획 업데이트](docs/aws-deployment/step-88-capacity-planning-update.md) 결과가 확보되어야 한다.
- 이해관계자 요구사항(1단계)와 제품 전략이 최신 상태여야 한다.

## 필요한 입력 자료
- 제품 백로그, 고객/사용자 피드백, SLA/SLO 성과 데이터.
- 예산, 인력 가용성, 기술 부채 목록.
- 외부 규제 변경, 시장 동향, 경쟁사 분석 자료.

## 상세 절차
1. **현황 분석**
   - KPI(90단계), 비용/성능 데이터(86·88단계), 포스트모템(96단계) 결과를 종합 평가한다.
   - 로드맵 상 미완료 항목과 신규 요구사항을 분류한다.
2. **우선순위 재정의**
   - 고객 가치, 위험 완화, 비용 절감, 전략적 중요도 기준으로 우선순위를 재정렬한다.
   - 97단계 회의에서 합의한 항목을 검토하고 필요시 재조정한다.
3. **로드맵 초안 작성**
   - 분기별/반기별 마일스톤을 설정하고 주요 산출물과 선행조건을 명시한다.
   - 인력/예산 배정을 계획하고 6단계 예산 한도와 일치하도록 조정한다.
4. **검토 및 승인**
   - 제품 오너, 기술 리더, 운영팀과 워크숍을 진행해 로드맵 초안을 검토한다.
   - 34단계 승인 프로세스에 따라 최종 승인 및 공지를 진행한다.
5. **커뮤니케이션 및 추적**
   - 로드맵을 프로젝트 관리 도구에 업데이트하고 이해관계자에게 공유한다.
   - 진행 상황을 97단계 정기 회의에서 추적하고 변동 사항을 반영한다.

## 의사결정 포인트
- 로드맵 주기를 분기/반기/연 단위 중 무엇으로 할지.
- 리소스 제약이 있을 때 어떤 항목을 연기하거나 축소할지.
- 신규 기술 도입(IaC 확장, 멀티 리전 등)의 우선순위 결정.

## 체크리스트
- [ ] 최신 KPI, 비용, 용량 데이터를 분석했다.
- [ ] 우선순위 기준과 결과가 이해관계자와 합의되었다.
- [ ] 로드맵 초안이 작성되고 승인 절차를 통과했다.
- [ ] 로드맵이 공유되고 추적 체계가 업데이트되었다.
