# 14단계. 네트워크 설계 초안 작성

## 목적
- AWS 네트워크 토폴로지(VPC, 서브넷, 게이트웨이 등)의 초안을 작성하여 서비스 연결성과 보안을 보장한다.
- 이후 세부 네트워크 구현 단계의 기준 문서를 마련한다.

## 선행조건
- 2단계 애플리케이션 아키텍처 다이어그램.
- 5단계 환경 구분 전략과 7단계 보안 요구사항.
- 11단계 계정 구조(OU/계정) 확정 결과.

## 필요한 입력 자료
- 서비스별 트래픽 패턴 및 네트워크 대역폭 요구사항.
- 온프레미스/타 클라우드 연동 필요성 및 연결 옵션.
- 고가용성, DR, 멀티 리전 요구사항.

## 상세 절차
1. **요구사항 수집 및 정리**
   - 애플리케이션별 연결 필요성, 포트, 프로토콜을 목록화한다.
   - 보안/규제 요구에 따른 네트워크 분리(예: PCI, 개인정보) 요구를 확인한다.
2. **VPC 및 서브넷 구조 초안**
   - 계정/환경별 VPC 수량, 리전 선택, AZ 분산 전략을 설계한다.
   - 서브넷 유형(퍼블릭/프라이빗/내부 전용)과 역할을 정의한다.
3. **네트워크 게이트웨이 및 라우팅**
   - Internet Gateway, NAT Gateway, Transit Gateway, VPN 등 필요한 게이트웨이를 식별한다.
   - 라우팅 테이블 구성을 설계하고, 네트워크 경로를 다이어그램으로 표현한다.
4. **보안 경계 정의**
   - 보안 그룹, NACL, Firewall Manager 등 레이어별 보안 제어 지점을 식별한다.
   - 로그 수집 지점(Flow Logs 등)과 모니터링 전략을 설정한다.
5. **고가용성/확장 계획**
   - AZ 장애 대비 방안과 오토스케일링, 로드밸런서 구성 계획을 수립한다.
   - 멀티 리전 배포 필요성을 검토하고 데이터 동기화 전략을 문서화한다.
6. **검토 및 검증**
   - 아키텍트, 보안팀, 네트워크 팀과 설계를 리뷰하여 피드백을 반영한다.
   - 15~17단계 세부 계획 작성에 필요한 TODO 목록을 도출한다.
7. **문서화 및 승인**
   - 네트워크 다이어그램, 설명 문서, 의사결정 기록을 저장하고 승인 절차를 진행한다.

## 의사결정 포인트
- 리전 및 AZ 선택 기준.
- Transit Gateway/Direct Connect 등 고급 네트워킹 서비스 사용 여부.

## 체크리스트
- [ ] VPC/서브넷 구조도와 설명 문서가 작성되었다.
- [ ] 라우팅 및 게이트웨이 구성이 정의되었다.
- [ ] 보안 경계 및 로깅 전략이 포함되었다.
- [ ] 이해관계자 리뷰 및 승인이 완료되었다.
