# 60단계. 애플리케이션 설정 값 검증

## 목적
- 52단계에서 준비한 설정 파일과 시크릿이 배포 환경에 정확히 주입되어 애플리케이션이 기대대로 동작하는지 확인한다.
- 설정 변경 시 발생할 수 있는 오류를 사전에 탐지하고 67단계 기능 검증에 필요한 기준선을 확보한다.

## 선행조건
- [52단계 애플리케이션 설정 파일 준비](docs/aws-deployment/step-52-application-config-prep.md).
- [45단계 빌드 시크릿 주입](docs/aws-deployment/step-45-build-secrets-injection.md) 및 [51단계 구성 관리 계획](docs/aws-deployment/step-51-configuration-management-plan.md).
- 58·59단계 모니터링/로그 보존 설정.

## 필요한 입력 자료
- 환경별 설정 템플릿, 시크릿 저장소 값.
- 애플리케이션 실행 로그, 헬스체크 엔드포인트, 구성 검증 스크립트.
- QA 테스트 케이스 및 요구사항 문서.

## 상세 절차
1. **배포 환경 선택 및 준비**
   - 개발/QA 환경에 최신 IaC 및 구성 관리 변경을 적용한다.
   - 56단계 빌드 결과물을 사용하여 애플리케이션을 배포한다.
2. **설정 값 주입 확인**
   - 애플리케이션 런타임에서 사용 중인 환경 변수, 설정 파일을 덤프하거나 API를 통해 확인한다.
   - 시크릿이 암호화되지 않은 형태로 로그에 노출되지 않는지 검토한다.
3. **기능/통합 테스트 실행**
   - 설정 값에 의존하는 기능(외부 API 호출, DB/캐시 연결 등)을 테스트한다.
   - 실패 시 어떤 설정 값이 잘못되었는지 추적한다.
4. **모니터링 및 로그 검증**
   - 58단계에서 구성한 대시보드/알람이 정상 데이터를 수집하는지 확인한다.
   - 59단계 보존 정책에 따라 로그가 올바른 위치로 전송되는지 검증한다.
5. **자동화 스크립트/테스트 케이스 업데이트**
   - 설정 검증을 자동화하기 위한 스크립트 또는 CI 테스트를 작성한다.
   - 63단계 CI 리허설에 포함하여 회귀 테스트를 구현한다.
6. **결과 기록 및 승인**
   - 확인된 설정 값, 테스트 결과, 발견된 이슈를 문서화한다.
   - 34단계 승인 절차에 따라 설정 변경 승인을 획득한다.

## 의사결정 포인트
- 설정 검증을 어느 단계에서 자동화할지(CI, CD, 운영 헬스체크 등).
- 비밀 정보 검증 시 샘플링/마스킹 전략.
- 설정 변경 승인 권한 및 승격 절차.

## 체크리스트
- [ ] 모든 주요 설정 값이 기대값과 일치함을 확인했다.
- [ ] 시크릿 노출 없이 로그/모니터링이 정상 동작한다.
- [ ] 자동화된 검증 스크립트/테스트가 파이프라인에 통합되었다.
- [ ] 검증 결과와 승인 기록이 문서화되었다.

## 산출물 및 보관 위치
- 설정 검증 스크립트 (`scripts/validate-config.sh` 등).
- 테스트 결과 보고서 (`docs/reports/config-validation.md`).
- 승인 기록(변경 관리 티켓, PR 링크 등).
