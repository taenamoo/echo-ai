# 59단계. 로그 수집 및 보존 정책 적용

## 목적
- 7단계 규제 요구사항과 37단계 태깅 정책을 준수하도록 로그 수집, 보존, 접근 제어 정책을 구체적으로 적용한다.
- 58단계에서 구축한 모니터링/로깅 인프라가 운영 기준에 맞게 설정되었는지 검증한다.

## 선행조건
- [58단계 모니터링/로깅 인프라 생성](docs/aws-deployment/step-58-monitoring-logging-provisioning.md).
- [7단계 보안 및 규제 요구사항 분석](docs/aws-deployment/step-07-security-compliance.md)과 [8단계 거버넌스 원칙](docs/aws-deployment/step-08-governance-principles.md).
- 36단계 비용 모니터링, 38단계 시크릿 운영 정책.

## 필요한 입력 자료
- 규제/내부 정책에서 요구하는 로그 유형 및 보존 기간.
- KMS 키, 접근 제어 목록, 데이터 분류 등급.
- 로그 아카이빙/삭제 프로세스 문서.

## 상세 절차
1. **로그 카탈로그 검토**
   - 애플리케이션, 인프라, 보안 로그 유형을 목록화하고 필수/선택 로그를 구분한다.
   - 태그/메타데이터(서비스, 환경, 민감도)를 정의한다.
2. **보존 및 아카이빙 정책 적용**
   - CloudWatch Logs 보존 기간, S3 Glacier 전환 규칙 등을 설정한다.
   - 규제 요구사항에 따라 최소/최대 보존 기간을 검증한다.
3. **접근 제어 구성**
   - IAM 정책, Lake Formation, Access Analyzer 등을 사용해 최소 권한 접근을 구현한다.
   - 접근 로그를 CloudTrail/Config에 기록하고 76단계 감사 검토 시 참조하도록 한다.
4. **무결성 및 감사 추적 확보**
   - 로그 무결성 검증(예: CloudTrail 로그 파일 검증, AWS Glue ETL 해시)을 구성한다.
   - 삭제/수정 시 승인 절차와 감사 로그 남기는 방식을 정의한다.
5. **비용 관리 및 최적화**
   - 보존 정책 변경이 비용에 미치는 영향을 분석하고 71단계 비용 알람에 반영한다.
   - 샘플링, 필터링, 압축 등 최적화 전략을 검토한다.
6. **운영 절차 문서화**
   - 로그 검색, 접근 요청 처리, 삭제 승인 절차를 Runbook에 기록한다.
   - 74단계 운영 핸드오프와 81단계 준수 감사 자료로 제공한다.

## 의사결정 포인트
- 장기 보존을 위한 아카이브 위치(S3 Glacier, 외부 보관 등).
- 민감 로그에 대한 추가 암호화 또는 토큰화 적용 여부.
- 접근 요청 승인 체계(보안팀 vs. 서비스팀)와 SLA.

## 체크리스트
- [ ] 로그 유형별 보존 기간이 정책과 일치한다.
- [ ] 접근 제어 및 감사 로깅이 구성되었다.
- [ ] 비용 영향이 분석되고 모니터링 체계에 반영되었다.
- [ ] 운영/감사 문서가 작성되었다.

## 산출물 및 보관 위치
- 로그 보존 정책 문서 (`docs/policies/log-retention.md`).
- 접근 제어 설정(IaC, IAM 정책) (`infra/logging/iam/`).
- 감사/검증 보고서 (`docs/reports/log-retention-audit.md`).
