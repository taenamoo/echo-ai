# 10단계. AWS 계정 생성 또는 온보딩

## 목적
- 프로젝트에 필요한 AWS 계정을 생성하거나 기존 계정에 대한 접근 권한을 정식으로 획득한다.
- 계정 보안 설정과 기본 거버넌스 구성을 완료하여 이후 인프라 작업의 기반을 마련한다.

## 선행조건
- 5단계 환경 정의서(계정 분리 전략) 및 8단계 거버넌스 정책.
- 7단계 보안 요구사항에 따른 필수 보안 통제 목록.
- 9단계 로드맵에서 계정 생성 일정이 승인되어 있어야 한다.

## 필요한 입력 자료
- AWS Organizations 또는 Control Tower 설정 가이드.
- 계정 생성 요청 양식, 예산 코드, 결제 수단 정보.
- IAM 역할 템플릿, SSO 그룹 매핑 표.

## 상세 절차
1. **계정 생성/온보딩 계획 확인**
   - 멀티 계정 전략에 따라 생성해야 할 계정 목록과 목적을 검토한다.
   - 기존 계정 온보딩 시 현재 설정과 정책 준수 여부를 사전 점검한다.
2. **계정 생성 요청 제출**
   - AWS Organizations 관리 계정 또는 Control Tower Account Factory를 통해 계정을 생성한다.
   - 재무/보안 승인 절차에 따라 계정 생성 요청 양식을 제출한다.
3. **기본 보안 설정 적용**
   - 루트 계정 MFA 활성화, 강력한 비밀번호 정책 설정.
   - CloudTrail 조직 트레일, AWS Config, GuardDuty, Security Hub를 활성화한다.
4. **SSO 및 IAM 역할 구성**
   - 1단계 이해관계자 정보와 8단계 거버넌스 정책에 따라 역할을 생성한다.
   - AWS IAM Identity Center 혹은 SAML 연동을 통해 접근 권한을 설정한다.
   - 서비스 계정/배포 도구용 IAM 역할을 생성하고 최소 권한 정책을 부여한다.
5. **네트워크/로그 기본 구성**
   - 계정별 기본 VPC 삭제 여부 및 신규 VPC 프로비저닝 계획을 확정한다.
   - 중앙 로그 아카이브 계정과 CloudTrail/Config 로그 S3 버킷 연계를 설정한다.
6. **예산 및 알람 설정**
   - 6단계에서 정의한 예산 한도에 따라 계정별 AWS Budgets를 설정한다.
   - 비용 및 보안 이벤트 알람을 구성한다.
7. **검증 및 문서화**
   - 체크리스트를 기반으로 계정 설정이 정책과 일치하는지 검증한다.
   - 계정 온보딩 보고서를 작성하고 승인자 서명을 받는다.

## 의사결정 포인트
- Control Tower를 사용할 경우 Landing Zone 구성 범위(공통 계정, 공통 서비스 등).
- 계정 간 네트워크 연결 방식을 즉시 설정할지 후속 단계에서 진행할지.

## 체크리스트
- [ ] 계정 생성 또는 온보딩 요청이 승인되었다.
- [ ] 루트 계정 MFA 및 기본 보안 서비스가 활성화되었다.
- [ ] IAM 역할 및 SSO 구성이 정책에 맞게 설정되었다.
- [ ] 예산 및 모니터링 알람이 구성되었다.
- [ ] 계정 온보딩 보고서가 작성되어 저장되었다.

## 산출물 및 보관 위치
- 계정 생성/온보딩 보고서 (예: `docs/accounts/account-onboarding-report.md`).
- IAM 역할 구성 파일(IaC 코드 또는 JSON 정책).
- 보안/비용 알람 설정 증적.
