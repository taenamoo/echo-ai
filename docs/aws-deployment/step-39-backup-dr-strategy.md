# 39단계. 백업 및 DR 전략 확정

## 목적
- 데이터 및 서비스 중단에 대비한 백업·재해 복구(Disaster Recovery) 전략을 확정한다.
- RPO/RTO 목표를 달성하기 위한 기술적/운영적 절차를 문서화한다.

## 선행조건
- 7단계 규제 요구사항, 23단계 데이터베이스 계획, 25단계 스토리지 전략.
- 26단계 모니터링, 33단계 롤백 정책, 38단계 비밀 관리 절차.
- 32단계 배포 전략에서 정의한 가용성 목표.

## 필요한 입력 자료
- 시스템 중요도 분류(미션 크리티컬, 중요, 일반) 및 의존성.
- 데이터 보호 요구(RPO/RTO, 보존 기간, 암호화 정책).
- 복구 대상 리소스 목록과 운영팀 조직도.

## 상세 절차
1. **복구 요구사항 정교화**
   - 서비스/데이터별 RPO, RTO, RCO(복구 신뢰도) 목표를 확정한다.
   - 0단계 요구사항 문서와 7단계 규제 조건을 재검토한다.
2. **백업 전략 정의**
   - DB: 자동 백업, 스냅샷, PITR, 교차 리전 복제 설정을 계획한다.
   - 스토리지: S3 버전 관리, Object Lock, Glacier 보관 정책을 정의한다.
   - 시크릿/구성: 38단계 절차에 따라 암호화된 백업을 준비한다.
3. **DR 아키텍처 설계**
   - 온사이트, 핫/웜/콜드 스탠바이, 멀티리전 전략 중 선택한다.
   - 네트워크 요구(14~19단계)와 IAM 역할(12단계)을 포함한 DR 환경 구성을 설계한다.
4. **운영 프로세스 및 테스트 계획 수립**
   - 백업 실행 스케줄, 무결성 검증, 실패 알림 절차를 문서화한다.
   - 정기 DR 연습(연 1회 이상) 시나리오와 성공 기준을 정의한다.
5. **비용 및 거버넌스 검토**
   - 6단계 예산과 36단계 비용 모니터링 계획에 DR 비용을 반영한다.
   - 감사 증적(로그, 리포트) 보관 위치와 접근 제어를 설정한다.

## 의사결정 포인트
- DR 전략 유형(핫/웜/콜드) 선택과 투자 수준.
- 백업 암호화 키 관리 방식(KMS CMK vs. 외부 HSM).

## 체크리스트
- [ ] RPO/RTO 목표와 서비스 우선순위가 확정되었다.
- [ ] 백업 및 복제 설정 계획이 문서화되었다.
- [ ] DR 아키텍처와 테스트 일정이 정의되었다.
- [ ] 비용/감사 요구 반영이 검토되었다.
