# 74단계. 운영 핸드오프 계획 수립

## 목적
- 개발팀에서 운영팀으로 서비스 책임을 이전하기 위한 인수인계 범위, 일정, 교육 계획을 수립한다.
- 운영팀이 배포, 모니터링, Incident 대응을 자율적으로 수행할 수 있도록 필요한 권한과 자료를 정리한다.

## 선행조건
- [73단계 문서화 정비](docs/aws-deployment/step-73-documentation-refresh.md) 완료.
- [58단계 모니터링 인프라](docs/aws-deployment/step-58-monitoring-logging-provisioning.md), [79단계 Incident 연동](docs/aws-deployment/step-79-incident-management-integration.md) 준비 상황 파악.
- [65단계 최초 배포](docs/aws-deployment/step-65-initial-dev-deployment.md)와 운영 절차 Runbook 확보.

## 필요한 입력 자료
- 팀별 역할/책임(RACI) 매트릭스, 접속 권한 목록, 온콜 일정 템플릿.
- 교육 커리큘럼 초안, 실습 환경 정보, 평가 체크리스트.
- 운영 SLA/SLO 문서 및 서비스 지원 프로세스(ITSM 도구) 정의서.

## 상세 절차
1. **인수인계 범위 정의**
   - 운영팀이 맡을 서비스 영역(배포, 모니터링, 보안 대응, 비용 관리)을 명확히 구분한다.
   - 각 영역에 필요한 접근 권한과 도구를 매핑한다.
2. **일정 및 담당자 지정**
   - 인수인계 기간, 교육 세션 일정, 실습 일정을 계획한다.
   - 운영/개발 책임자와 승인자를 지정하고 커뮤니케이션 채널을 설정한다.
3. **교육 자료 및 실습 시나리오 준비**
   - 73단계 문서를 기반으로 교육 슬라이드, 데모 스크립트, 실습 가이드를 제작한다.
   - 배포 리허설, Incident 대응, 백업 복구 등 핵심 시나리오를 포함한다.
4. **권한/접근 제어 정비**
   - IAM 역할, SSO 그룹, 접근 권한을 검토하여 운영팀에 필요한 권한을 부여한다.
   - 불필요한 개발자 권한은 제거하고 감사 로그(76단계)와 정합성을 확인한다.
5. **검토 및 승인**
   - 운영팀과 개발팀이 계획서를 공동 검토하고 개선 사항을 반영한다.
   - 최종 계획에 경영진/프로젝트 매니저 승인을 받아 일정 확정 공지를 발송한다.

## 의사결정 포인트
- 운영팀이 배포 권한을 직접 수행할지, 개발팀 승인 후 실행할지 여부.
- 온콜/Incident 대응을 24x7로 운영할지, 근무 시간 내로 제한할지 여부.
- 교육을 온사이트로 진행할지, 온라인 세션 및 녹화 자료로 대체할지 여부.

## 체크리스트
- [ ] 인수인계 범위와 책임이 문서화되고 모든 이해관계자가 승인했다.
- [ ] 교육 일정, 실습 계획, 평가 기준이 확정되었다.
- [ ] 운영팀에 필요한 접근 권한이 부여되고 감사 로그에 기록되었다.
- [ ] 인수인계 완료 보고 및 후속 지원 계획이 수립되었다.
