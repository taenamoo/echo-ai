# 34단계. 배포 승인 절차 수립

## 목적
- 환경별 배포 전 필요한 승인 체계를 정의하여 변경 관리 거버넌스를 확보한다.
- 승인 흐름을 CI/CD 파이프라인과 운영 프로세스에 통합한다.

## 선행조건
- 1단계 이해관계자 매핑 결과(RACI).
- 27단계 CI/CD 요구사항과 31단계 품질 게이트 기준.
- 32단계 배포 전략과 33단계 롤백 정책 문서.

## 필요한 입력 자료
- 규제 준수 요구사항 및 변경 관리 정책(7, 8단계).
- 팀별 업무 시간, 온콜 스케줄, 커뮤니케이션 채널 정보.
- 배포 환경(개발/스테이징/운영)별 위험도 평가.

## 상세 절차
1. **승인 역할 정의**
   - 환경별로 승인자(개발 리드, QA, 보안, 운영)를 지정한다.
   - 1단계 RACI 매트릭스를 업데이트하여 책임과 권한을 명확히 한다.
2. **승인 기준 수립**
   - 31단계 스캔 결과, 32단계 배포 전략 성공 기준, 33단계 롤백 준비 상태를 승인 체크리스트에 포함한다.
   - 릴리스 노트, 변경 영향도 분석, 테스트 증적 요구사항을 정의한다.
3. **파이프라인 통합**
   - 28~29단계에서 선택한 CI/CD 도구에 수동 승인 게이트(Manual Approval, ChatOps 등)를 구성한다.
   - 승인 요청/완료 기록을 감사 목적으로 보관할 저장소를 지정한다.
4. **커뮤니케이션 및 알림 흐름 구성**
   - 슬랙/이메일/이슈 트래커 등으로 승인 요청 알림을 자동화한다.
   - 승인 지연 시 에스컬레이션 경로를 정의한다.
5. **운영 문서화 및 교육**
   - 승인 절차, 체크리스트, 예외 처리 방법을 Runbook으로 정리한다.
   - 신규 팀원 온보딩 시 교육 계획을 수립한다.

## 의사결정 포인트
- 환경별 승인자 수 및 단계(단일 vs. 다중 승인).
- 긴급 변경 시 예외 승인 절차 및 승인 기록 방식.

## 체크리스트
- [ ] 승인 역할과 책임이 문서화되었다.
- [ ] 승인 기준/체크리스트가 합의되었다.
- [ ] CI/CD 파이프라인 승인 게이트가 구성되었다.
- [ ] 승인 기록 저장 및 감사 절차가 정의되었다.

## 산출물 및 보관 위치
- 배포 승인 정책 문서 (`docs/aws-deployment/deployment-approval-process.md`).
- 파이프라인 승인 설정(`infra/pipelines/approvals/`).
- 승인 기록/감사 로그 저장 경로 안내 (`docs/runbooks/deployment-approvals.md`).
