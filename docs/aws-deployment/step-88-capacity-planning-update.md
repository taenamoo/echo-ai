# 88단계. 성능 및 용량 계획 업데이트

## 목적
- 실제 운영 데이터를 기반으로 성능 및 용량 계획을 갱신하여 향후 트래픽 변화에 대비한다.
- 69단계 성능 테스트 결과와 77단계 자동 스케일링 정책을 검증하고 조정한다.

## 선행조건
- [68단계 성능 테스트 계획](docs/aws-deployment/step-68-performance-test-planning.md)과 [69단계 성능 테스트 실행](docs/aws-deployment/step-69-performance-test-execution.md) 결과가 정리되어 있어야 한다.
- [77단계 자동 스케일링 정책](docs/aws-deployment/step-77-auto-scaling-policy.md)이 배포되어 있어야 한다.
- [83단계 운영 대시보드](docs/aws-deployment/step-83-operations-dashboard.md)에서 수집한 실사용 데이터가 확보되어야 한다.

## 필요한 입력 자료
- 최근 트래픽/성능 메트릭(응답시간, CPU, 메모리, I/O 등).
- 사용자 성장/릴리즈 계획(9단계 로드맵) 및 마케팅 캠페인 일정.
- 비용 데이터(86단계)와 SLA/SLO 목표(84단계).

## 상세 절차
1. **현황 분석**
   - 운영 데이터와 테스트 결과를 비교하여 현재 성능 여유와 병목을 파악한다.
   - 83단계 대시보드에서 피크 시간대를 확인하고 이상 지표를 기록한다.
2. **수요 예측**
   - 비즈니스 계획, 마케팅 일정, 사용자 성장률을 반영하여 향후 트래픽 시나리오를 작성한다.
   - 계절성, 이벤트 등을 고려해 베스트/워스트 케이스를 정의한다.
3. **용량 모델 갱신**
   - 컴퓨팅, 데이터베이스, 캐시, 네트워크 리소스별 용량 산정을 업데이트한다.
   - 스케일링 정책(77단계)과 일치하도록 임계값, 최소/최대 용량을 조정한다.
4. **개선 계획 수립**
   - 용량 확장, 최적화, 캐시 전략 변경 등 필요한 조치를 정의하고 일정/비용을 추정한다.
   - 변경이 필요한 경우 IaC/배포 파이프라인(61단계 이후)을 업데이트한다.
5. **검토 및 승인**
   - 이해관계자와 리뷰 회의를 진행하여 계획을 승인받고 90단계 KPI에 관련 지표를 반영한다.
   - 변경 사항을 95단계 배포 기록과 98단계 로드맵에 업데이트한다.

## 의사결정 포인트
- 용량 계획의 리뷰 주기(월간/분기/반기) 결정.
- 온디맨드 vs. 예약 인스턴스/세이빙 플랜 등 비용 전략 조정 여부.
- 수요 급증 시 수동 확장 vs. 자동 스케일링만으로 대응할지 결정.

## 체크리스트
- [ ] 최신 운영 데이터와 테스트 결과가 비교 분석되었다.
- [ ] 향후 수요 시나리오와 용량 모델이 업데이트되었다.
- [ ] 개선 계획이 승인되고 실행 일정이 확정되었다.
- [ ] 변경 사항이 관련 문서와 지표에 반영되었다.
