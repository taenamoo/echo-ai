# 17단계. 보안 그룹 및 네트워크 ACL 정책 수립

## 목적
- AWS 네트워크 보안 계층(보안 그룹, 네트워크 ACL 등)의 정책을 정의하여 최소 권한 원칙을 적용한다.
- 애플리케이션 요구와 규제 요건에 맞는 트래픽 제어 기준을 문서화한다.

## 선행조건
- 14단계 네트워크 설계 초안과 15단계 CIDR 계획.
- 16단계 VPC/서브넷 생성 계획.
- 7단계 보안 요구사항 및 12단계 IAM 설계.

## 필요한 입력 자료
- 애플리케이션별 포트/프로토콜 요구사항.
- 규제/감사 기준에서 요구하는 네트워크 통제 정책.
- 보안 모니터링 및 침입 탐지 시스템 요구사항.

## 상세 절차
1. **트래픽 흐름 분석**
   - 서비스 간 통신, 외부 사용자 접근, 관리용 접속 시나리오를 식별한다.
   - 데이터 분류 수준에 따라 필요한 네트워크 분리를 검토한다.
2. **보안 그룹 설계**
   - 각 서비스 또는 컴포넌트에 필요한 인바운드/아웃바운드 규칙을 정의한다.
   - 태그 기반 규칙 적용 여부와 기본 거부 정책을 검토한다.
3. **네트워크 ACL 설계**
   - 서브넷 수준에서 적용할 허용/거부 규칙을 정의하고 순서를 문서화한다.
   - 로그 수집(Flow Logs) 및 감사 요구를 포함한다.
4. **자동화 및 형상 관리 전략**
   - IaC 또는 AWS Firewall Manager를 사용한 정책 배포 방법을 정의한다.
   - 정책 변경 시 승인 및 리뷰 절차를 설정한다.
5. **검증 및 테스트 계획**
   - 테스트 케이스(포트 스캔, 연결 시도)를 작성하고 도구를 선정한다.
   - 비상 상황을 위한 임시 예외 처리 절차를 마련한다.
6. **운영 가이드 작성**
   - 변경 이력 관리, 모니터링, 경보 대응 방법을 포함한 운영 매뉴얼을 작성한다.
   - 정기적 규칙 검토 및 갱신 주기를 정의한다.
7. **승인 및 배포 준비**
   - 보안팀, 네트워크팀, 애플리케이션 팀의 승인을 받아 정책을 확정한다.
   - 배포 일정과 커뮤니케이션 계획을 수립한다.

## 의사결정 포인트
- 중앙화된 Firewall Manager 도입 여부.
- 기본 거부(Default Deny) 정책 적용 범위 및 예외 처리 프로세스.

## 체크리스트
- [ ] 서비스별 보안 그룹 규칙표가 작성되었다.
- [ ] 서브넷별 NACL 정책과 로그 수집 계획이 정의되었다.
- [ ] 정책 배포 자동화 및 변경 관리 절차가 수립되었다.
- [ ] 검증 계획과 승인 문서가 완료되었다.
