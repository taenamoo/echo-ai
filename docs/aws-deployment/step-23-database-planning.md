# 23단계. 데이터베이스 선택 및 프로비저닝 계획

## 목적
- 애플리케이션 요구사항에 적합한 AWS 데이터베이스 서비스를 선정하고 초기 프로비저닝 전략을 수립한다.
- 가용성, 확장성, 보안, 백업 요구사항을 충족하는 데이터 계층 설계안을 마련한다.

## 선행조건
- 0단계 요구사항 명세, 2단계 아키텍처, 7단계 보안 요구사항.
- 6단계 비용 계획, 20단계 컴퓨팅 서비스 결정.
- 22단계 시크릿 관리 전략.

## 필요한 입력 자료
- 데이터 모델(관계형/비관계형), 예상 데이터량, 성능 지표(RPS, 레이턴시).
- 복제, DR(Disaster Recovery), 백업 주기 및 보존 기간.
- 트랜잭션 일관성, 멀티리전 요구사항, 규제 준수 항목.

## 상세 절차
1. **워크로드 특성 분석**
   - 읽기/쓰기 비율, 트랜잭션 특성, 스키마 복잡도를 검토한다.
   - OLTP, 분석, 시계열 등 패턴을 식별한다.
2. **서비스 후보 평가**
   - RDS(MySQL/PostgreSQL), Aurora, DynamoDB, DocumentDB, Redshift 등 후보를 나열한다.
   - 관리형 vs. 자체 구축(EC2) 옵션을 비용·운영 관점에서 비교한다.
3. **아키텍처 및 스토리지 설계**
   - Multi-AZ, Read Replica, Global Database, DynamoDB Global Table 등 고가용성 옵션을 평가한다.
   - 스토리지 계층(S3 백업, EBS 볼륨 유형)을 결정한다.
4. **보안 및 접근 통제 설계**
   - 네트워크 경계(VPC/Subnet), 보안 그룹, 파라미터 그룹을 정의한다.
   - 암호화(KMS), 인증 방식, 감사 로깅을 설계한다.
5. **백업 및 복구 전략 수립**
   - 스냅샷 스케줄, PITR(Point-in-Time Recovery), 테스트 복구 절차를 문서화한다.
   - 6단계 예산에 백업 저장 비용을 반영한다.
6. **마이그레이션 및 초기 로드 계획**
   - DMS, 스키마 변환, 초기 데이터 시드 절차를 정의한다.
   - 다운타임, 데이터 검증 방법, 롤백 방안을 마련한다.
7. **IaC 및 자동화 설계**
   - 4단계에서 결정한 IaC 도구로 데이터베이스 리소스를 정의한다.
   - 파라미터 설정, 초기 사용자 생성, 모니터링 설정을 자동화한다.

## 의사결정 포인트
- 완전관리형 서비스(Aurora, DynamoDB) vs. 자체 운영.
- 단일 리전/계정 vs. 멀티 리전/크로스 계정 배치.

## 체크리스트
- [ ] 데이터베이스 서비스 및 규모가 결정되었다.
- [ ] 보안, 백업, DR 계획이 문서화되었다.
- [ ] IaC 템플릿과 초기 프로비저닝 Runbook이 준비되었다.
- [ ] 마이그레이션/로드 전략과 검증 절차가 확정되었다.
