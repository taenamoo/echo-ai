# 29단계. CD 도구 선택

## 목적
- 선택된 컴퓨팅 환경에 애플리케이션을 배포하기 위한 지속적 배포(CD) 도구를 선정한다.
- 배포 전략, 승인 절차, 롤백 요구사항을 자동화할 수 있는 플랫폼을 결정한다.

## 선행조건
- 20단계 컴퓨팅 서비스 결정, 27단계 파이프라인 요구사항 정의.
- 28단계 CI 도구 선택, 32~33단계 배포/롤백 전략 초안(선행 검토).
- 7단계 보안, 8단계 거버넌스 정책.

## 필요한 입력 자료
- 환경별 배포 빈도, 가동 중지 허용 범위, 승인 프로세스.
- IaC/애플리케이션 배포 방식(blue/green, canary, rolling).
- 접근 제어, 감사, 멀티 계정/리전 지원 요구사항.

## 상세 절차
1. **후보 도구 정의**
   - AWS CodeDeploy, CodePipeline, Argo CD, Spinnaker, Flux, Harness 등.
   - 서버리스, 컨테이너, VM 배포 지원 여부를 고려한다.
2. **배포 전략 지원성 평가**
   - Blue/Green, Canary, 자동 롤백, 승인 게이트, 배포 전후 훅 지원 여부를 확인한다.
   - 멀티 환경 프로모션, 매뉴얼 승인 통합 가능성을 평가한다.
3. **아키텍처 및 운영 모델 검토**
   - 관리형 서비스 vs. 자체 호스팅 컨트롤 플레인.
   - 고가용성, 확장성, 백업/복구 절차를 검토한다.
4. **보안 및 접근 제어 평가**
   - IAM 역할 위임, 시크릿 연동(22단계), 감사 로그 제공 여부를 확인한다.
   - 멀티 계정/리전 배포 시 Cross-account 접근 전략을 검토한다.
5. **비용 및 라이선스 분석**
   - 사용량 기반 요금, 라이선스 비용, 운영 인력 투입량을 계산한다.
6. **PoC 실행 및 통합 검증**
   - 대표 서비스에 대한 테스트 배포를 수행하고 롤백/승인 시나리오를 검증한다.
   - 26단계 모니터링과 알림 통합을 테스트한다.
7. **최종 결정 및 문서화**
   - 선택 사유, 비교 결과, 제약 조건을 정리한다.
   - 운영 책임자, 접근 통제, 변경 절차를 명시한다.

## 의사결정 포인트
- SaaS/관리형 vs. 자체 호스팅 CD 플랫폼.
- GitOps(Argo CD/Flux) vs. 파이프라인 기반(CodeDeploy/Spinnaker) 접근.

## 체크리스트
- [ ] 후보 도구 비교 및 배포 전략 지원성 평가가 완료되었다.
- [ ] 보안, 멀티 환경, 비용 요구사항을 충족하는 도구가 선정되었다.
- [ ] PoC에서 롤백/승인/모니터링 연동이 검증되었다.
- [ ] 운영 모델과 책임이 문서화되었다.
