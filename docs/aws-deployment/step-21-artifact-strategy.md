# 21단계. 이미지 및 아티팩트 전략 수립

## 목적
- 애플리케이션 빌드 산출물(컨테이너 이미지, 패키지, 번들 등)을 어디에 저장하고 관리할지 결정한다.
- 다중 환경 배포를 지원할 수 있는 버저닝, 보안, 수명 주기 정책을 수립한다.

## 선행조건
- 3단계 버전 관리 전략과 4단계 IaC 결정.
- 6단계 비용 한도, 7단계 보안·규제 요구사항.
- 20단계 컴퓨팅 서비스 선택 결과.

## 필요한 입력 자료
- 애플리케이션 빌드 산출물 유형과 크기 추정치.
- 보안/규제 요구사항(이미지 서명, 취약점 스캔, 보존 기간 등).
- 멀티 리전 또는 하이브리드 배포 여부.

## 상세 절차
1. **아티팩트 유형 분류**
   - 컨테이너 이미지, 함수 패키지, 정적 웹 에셋, 라이브러리 패키지를 식별한다.
   - 각 유형별 생성 주기와 대상 환경(개발/스테이징/운영)을 정의한다.
2. **저장소 옵션 평가**
   - 컨테이너: Amazon ECR, EKS/ECS 프라이빗 레지스트리, 하이브리드 시 Harbor 등.
   - 패키지: AWS CodeArtifact, S3, 외부 레지스트리(Nexus, Artifactory) 등을 비교한다.
   - 6단계 비용 한도, 운영 복잡도, 7단계 보안 요구를 기준으로 선택한다.
3. **네이밍 및 버전 규칙 정의**
   - Git 태그/커밋 기반 버저닝, 브랜치별 태그 규칙, 환경별 프리픽스를 결정한다.
   - 3단계 브랜치 전략과 정합성을 검토한다.
4. **접근 제어 및 네트워크 정책 설계**
   - IAM 정책, 리포지토리별 권한 모델(푸시/풀/관리)을 정의한다.
   - VPC 엔드포인트, 프라이빗 링크 사용 여부를 결정한다.
5. **라이프사이클 관리 및 복제 전략 수립**
   - 태그 기반 정리 정책, 이미지 만료, 장기 보관 정책을 정의한다.
   - 멀티 리전 배포 시 복제 정책(ECR 레플리케이션, S3 CRR)을 설정한다.
6. **보안·품질 게이트 결정**
   - 이미지 서명(예: Notary, cosign), 취약점 스캔, SBOM 생성 도구를 선택한다.
   - 31단계 보안 스캔 기준과 연동 가능하도록 요구사항을 정리한다.
7. **운영 절차 문서화**
   - 빌드 파이프라인(30단계)에서 아티팩트를 발행하고 검증하는 절차를 문서화한다.
   - 비상 상황 시 아티팩트 롤백 방법, 삭제 금지 정책을 명시한다.

## 의사결정 포인트
- 관리형 서비스(ECR/CodeArtifact) vs. 자체 호스팅 레지스트리.
- 지역 간 복제 및 오프라인 백업 필요 여부.

## 체크리스트
- [ ] 아티팩트 유형별 저장소와 접근 정책이 결정되었다.
- [ ] 버저닝 및 라이프사이클 전략이 문서화되었다.
- [ ] 보안/스캔 요구사항이 정의되었다.
- [ ] 운영 절차와 롤백 방안이 기록되었다.
