# 92단계. 보안 교육 및 인식 제고

## 목적
- 운영 및 개발 인력이 AWS 보안 모범 사례와 사고 대응 절차를 숙지하도록 교육한다.
- 75·81단계에서 정의한 보안 정책을 실무에 적용해 사고 가능성을 줄인다.

## 선행조건
- [75단계 보안 사고 대응 절차](docs/aws-deployment/step-75-security-incident-response.md)와 [81단계 보안 감사 결과](docs/aws-deployment/step-81-security-compliance-audit.md)가 정리되어 있어야 한다.
- 12단계 IAM 설계, 82단계 원격 접근 절차가 최신 상태로 공유되어야 한다.
- 온보딩 패키지(91단계)에 기본 보안 안내 초안이 포함되어 있어야 한다.

## 필요한 입력 자료
- 조직 보안 정책, 규제 준수 요구사항, 내부 통제 문서.
- 최근 보안 사고 사례, 침해 대응 시나리오, 모의 훈련(89단계) 결과.
- 교육 일정, 참석 대상 목록, 교육 플랫폼(온라인/오프라인) 정보.

## 상세 절차
1. **교육 대상 및 목표 정의**
   - 팀별로 필수 교육 수준을 정의하고 신규/기존 인력에 대한 교육 주기를 설정한다.
   - 7단계 규제 요구사항을 참고해 필수 포함 항목을 식별한다.
2. **교육 자료 준비**
   - IAM, 네트워크 보안, 시크릿 관리, 데이터 보호, 사고 대응 등 핵심 주제를 포함한다.
   - 81단계 감사에서 발견된 취약점과 개선 조치를 사례로 추가한다.
3. **교육 운영 계획 수립**
   - 일정, 강사, 플랫폼(사내 LMS, 화상 회의)을 결정하고 초대장을 발송한다.
   - 교육 전 퀴즈 또는 설문으로 사전 지식을 파악한다.
4. **교육 실시 및 참여 기록**
   - 교육 중 데모나 실습을 제공하고 질문을 수집한다.
   - 참석자 명단과 이해도 평가 결과를 기록해 94단계 변경 관리 기록과 연동한다.
5. **사후 평가 및 개선**
   - 교육 후 퀴즈/설문 결과를 분석해 이해도가 낮은 항목을 파악한다.
   - 개선 사항을 온보딩 자료(91단계)와 Runbook에 반영하고 다음 교육 계획에 포함한다.

## 의사결정 포인트
- 교육을 의무화할 대상과 선택 참석 대상을 구분하는 기준.
- 교육 형식(라이브 vs. 온디맨드), 언어, 자료 접근 권한 범위.
- 교육 완료를 배포 권한 부여 조건으로 연결할지 여부.

## 체크리스트
- [ ] 교육 대상과 목표가 정의되고 승인되었다.
- [ ] 최신 보안 정책과 감사 결과가 교육 자료에 반영되었다.
- [ ] 교육 일정이 공지되고 세션이 계획대로 진행되었다.
- [ ] 참석 기록과 평가 결과가 저장되고 후속 조치가 도출되었다.
