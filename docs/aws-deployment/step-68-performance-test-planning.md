# 68단계. 성능 테스트 계획 수립

## 목적
- 서비스 수준 목표(SLO)를 만족하기 위한 성능 테스트 전략과 지표를 정의한다.
- 69단계 실행을 위한 준비 자료(시나리오, 데이터, 도구)를 마련한다.

## 선행조건
- [26단계 모니터링 프레임워크](docs/aws-deployment/step-26-monitoring-logging-framework.md)와 [58단계 관측 인프라](docs/aws-deployment/step-58-monitoring-logging-provisioning.md)에서 지표 수집 체계가 갖춰져야 한다.
- [20단계 컴퓨팅 서비스 선택](docs/aws-deployment/step-20-compute-service-selection.md), [24단계 캐시/메시징 계획](docs/aws-deployment/step-24-cache-messaging-planning.md), [25단계 스토리지 전략](docs/aws-deployment/step-25-storage-strategy.md) 정보를 참고한다.
- [67단계 기능 검증](docs/aws-deployment/step-67-functional-validation.md)에서 핵심 기능이 안정적으로 동작해야 한다.

## 필요한 입력 자료
- 사용자 수요 예측, 피크 트래픽 데이터, SLA/SLO 문서.
- 테스트 도구 선택(JMeter, Locust, k6 등) 및 인프라 요구사항.
- 테스트 데이터 세트 및 시나리오 정의서.

## 상세 절차
1. **목표 지표 정의**
   - 응답 시간, 처리량, 오류율, 자원 사용량( CPU, 메모리, I/O ) 목표치를 설정한다.
   - 36단계 비용 모니터링과 연계하여 비용 임계값을 고려한다.
2. **테스트 유형 선정**
   - 부하 테스트, 스트레스 테스트, 장기 안정성 테스트 등 필요한 유형을 결정한다.
   - 각 테스트 유형의 종료 조건과 성공 기준을 명확히 한다.
3. **시나리오 및 데이터 설계**
   - 실제 사용자 행동을 반영한 시나리오를 작성하고 필요 데이터(계정, 주문, 메시지 등)를 준비한다.
   - 테스트 데이터 익명화/마스킹 정책을 38단계 시크릿 운영 지침과 일치시킨다.
4. **인프라 및 도구 준비 계획**
   - 테스트 실행 위치(VPC 내부, 외부 로드 제너레이터)와 권한을 정의한다.
   - 테스트 도구 설치/구성을 자동화하기 위한 IaC 또는 스크립트를 작성한다.
5. **모니터링 및 로깅 전략 수립**
   - 테스트 중 수집할 지표와 대시보드를 정의한다.
   - 알람 임계값 조정 계획을 마련하고 71단계 비용 알람과 충돌하지 않도록 한다.
6. **리스크 및 롤백 계획 수립**
   - 테스트로 인해 서비스에 미칠 영향과 완화 전략을 문서화한다.
   - 실패 시 리소스 해제, 설정 복구 절차를 정의한다.

## 의사결정 포인트
- 테스트를 dev 환경에서 수행할지, 별도 퍼포먼스 환경을 마련할지.
- 실제 사용자 데이터 사용 여부와 개인정보 보호 조치.
- 테스트 도구를 관리형 서비스로 사용할지 자체 호스팅할지.

## 체크리스트
- [ ] 성능 목표와 성공 기준이 문서화되었다.
- [ ] 테스트 시나리오, 데이터, 도구가 준비되었다.
- [ ] 모니터링 및 알람 전략이 정의되었다.
- [ ] 리스크 및 롤백 계획이 승인되었다.
