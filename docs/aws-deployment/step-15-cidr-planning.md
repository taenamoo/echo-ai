# 15단계. IP 대역 및 CIDR 계획 확정

## 목적
- AWS 네트워크에 사용할 IP 주소 대역과 CIDR 블록을 충돌 없이 설계하여 확장성과 호환성을 확보한다.
- 온프레미스, 타 클라우드, 파트너 네트워크와의 연동을 고려한 주소 체계를 정의한다.

## 선행조건
- 14단계 네트워크 설계 초안.
- 기존 온프레미스/타 클라우드 주소 계획 문서.
- 6단계 비용 계획(Transit Gateway, NAT 등 네트워크 비용 영향).

## 필요한 입력 자료
- 현재 사용 중인 IP 대역 목록 및 예약 대역.
- 미래 확장(추가 VPC, 리전) 계획과 예상 서비스 수요.
- 규제 또는 보안 정책에 따른 네트워크 분리 요구.

## 상세 절차
1. **현황 조사 및 겹침 분석**
   - 온프레미스, 데이터센터, 타 클라우드의 CIDR 목록을 수집한다.
   - 중복 가능성이 있는 대역을 식별하고 피해야 할 범위를 정의한다.
2. **주소 계획 원칙 수립**
   - 환경/계정/서비스별 주소 할당 규칙을 정의한다.
   - 확장성을 위해 예약 대역을 확보하고, 세그먼트 크기를 표준화한다.
3. **VPC/서브넷별 CIDR 배정**
   - 14단계에서 정의한 VPC 구조에 맞춰 CIDR을 배분한다.
   - 서브넷별 네트워크 마스크와 사용 가능한 IP 수를 계산하여 문서화한다.
4. **연동 네트워크 고려**
   - VPC 피어링, Transit Gateway, VPN 연결 시 충돌 방지 전략을 수립한다.
   - 온프레미스 라우팅 테이블 업데이트 계획을 작성한다.
5. **검증 및 시뮬레이션**
   - IPAM(AWS IP Address Manager) 또는 스프레드시트를 활용해 충돌 여부를 검증한다.
   - 향후 확장 시나리오(신규 서비스 추가)에서 문제가 없는지 검토한다.
6. **문서화 및 승인**
   - CIDR 할당 표, 라우팅 영향 분석, 변경 이력 기록을 작성한다.
   - 네트워크/보안팀, 인프라 아키텍트의 승인을 획득한다.

## 의사결정 포인트
- RFC1918 vs. 공인 IP 사용 여부 및 NAT 전략.
- AWS VPC IPAM 도입 여부와 운영 방식.
