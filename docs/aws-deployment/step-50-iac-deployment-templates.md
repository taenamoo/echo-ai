# 50단계. 배포 템플릿/IaC 코드 작성

## 목적
- 16단계 VPC 설계와 32단계 배포 전략을 반영하여 애플리케이션 배포에 필요한 IaC 템플릿(CloudFormation, Terraform, CDK 등)을 작성한다.
- CD 파이프라인(49단계)이 템플릿을 사용하여 일관된 방식으로 인프라 및 애플리케이션을 배포하도록 한다.

## 선행조건
- [16단계 VPC 및 서브넷 생성 계획서](docs/aws-deployment/step-16-vpc-subnet-implementation-plan.md).
- [29단계 CD 도구 선택](docs/aws-deployment/step-29-cd-tool-selection.md) 및 [49단계 CD 도구 설치](docs/aws-deployment/step-49-cd-tool-bootstrap.md).
- 37단계 태깅 규칙, 33단계 롤백 정책, 39단계 DR 전략.

## 필요한 입력 자료
- 애플리케이션 아키텍처 다이어그램(2단계)과 컴퓨팅 서비스 선택 결과(20단계).
- 보안 그룹, IAM 역할, 시크릿, 스토리지 구성 등 세부 요구사항.
- 배포 환경별 파라미터(네트워크, 용량, 확장 정책 등).

## 상세 절차
1. **템플릿 구조 설계**
   - 모듈/스택 구조를 정의하고 공통 모듈과 환경별 모듈을 분리한다.
   - 버전 관리 전략(3단계)에 맞춰 템플릿 저장소 브랜치 전략을 정한다.
2. **리소스 정의 작성**
   - 컴퓨팅 리소스(EC2/ECS/EKS/Lambda), 로드밸런서, 보안 그룹, IAM 역할 등을 코드로 정의한다.
   - 42단계에서 생성한 아티팩트 저장소, 45단계 시크릿을 참조하도록 구성한다.
3. **파라미터 및 변수 설계**
   - 환경(dev/test/prod)별 차이를 파라미터화하여 재사용성을 확보한다.
   - 기본값, 제약 조건, 설명을 포함하여 문서화한다.
4. **태깅 및 정책 적용**
   - 37단계 태깅 카탈로그를 모든 리소스에 적용한다.
   - 36단계 비용 모니터링과 연계될 수 있도록 Cost Center, Service 태그를 포함한다.
5. **검증 및 테스트**
   - IaC 도구의 검증 명령(`terraform validate`, `cfn-lint`, `cdk synth/test`)을 실행한다.
   - 64단계 드라이런 전에 Plan/Diff 결과를 리뷰하고 33단계 롤백 전략과 정합성을 확인한다.
6. **문서화 및 코드 리뷰**
   - 템플릿 구조, 파라미터 설명, 의존성을 문서화한다.
   - 코드 리뷰 절차를 거쳐 34단계 승인 정책을 준수한다.

## 의사결정 포인트
- 모놀리식 스택 vs. 마이크로 스택(서비스별 분리) 선택.
- IaC 코드 저장소를 앱 코드와 동일 저장소로 유지할지 분리할지 여부.
- 변경 관리(94단계)와 연동되는 릴리스 프로세스 정의.

## 체크리스트
- [ ] IaC 템플릿이 버전 관리 저장소에 커밋되었다.
- [ ] 필수 리소스와 태깅, 보안 정책이 코드에 반영되었다.
- [ ] 검증 도구 실행 및 코드 리뷰 기록이 남아 있다.
- [ ] 배포 파이프라인에서 템플릿을 참조하도록 설정되었다.

## 산출물 및 보관 위치
- IaC 코드 (`infra/iac/*`, `deployments/templates/*`).
- 파라미터 시트 및 환경 구성 파일 (`docs/config/deployment-parameters-dev.yaml`).
- 검증/리뷰 결과 (`docs/reports/iac-validation.md`).
