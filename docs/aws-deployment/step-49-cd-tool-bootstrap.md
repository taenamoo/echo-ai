# 49단계. CD 도구 설치 및 초기 설정

## 목적
- 29단계에서 선정한 CD 도구(CodeDeploy, Argo CD, Spinnaker 등)를 설치하거나 서비스에 등록하여 배포 자동화를 준비한다.
- 배포 대상 환경(dev)과 인증 체계를 구성해 이후 배포 스크립트(61단계)를 실행할 수 있도록 한다.

## 선행조건
- [29단계 CD 도구 선택](docs/aws-deployment/step-29-cd-tool-selection.md).
- [41단계 IAM 역할/사용자 생성 및 검증](docs/aws-deployment/step-41-iam-provisioning.md).
- 40단계 개발 환경 네트워크 구축, 42단계 아티팩트 저장소 생성.

## 필요한 입력 자료
- CD 도구 설치 가이드 또는 SaaS 설정 매뉴얼.
- 배포 대상 클러스터/인스턴스 정보(EC2, ECS, EKS 등)와 네트워크 엔드포인트.
- IAM 역할, 시크릿, 아티팩트 경로(ECR, S3) 등 인증 자료.

## 상세 절차
1. **배포 모델 및 아키텍처 확정**
   - 관리형 서비스(AWS CodePipeline/CodeDeploy) vs. 셀프 호스팅(Argo CD 등)을 결정한다.
   - 고가용성, 멀티 리전 요구사항을 검토하여 구성 방식을 문서화한다.
2. **인프라 배포 및 접속 구성**
   - IaC 또는 설치 스크립트로 CD 서버/컨트롤 플레인을 배포한다.
   - 40단계 네트워크에 접근할 수 있도록 VPC, 서브넷, 보안 그룹을 설정한다.
3. **인증 및 권한 설정**
   - 41단계 IAM 역할을 사용하여 AWS 리소스를 제어할 수 있도록 설정한다.
   - Kubernetes/ECS와 연동 시 서비스 어카운트, OIDC, IRSA 등의 권한을 구성한다.
4. **아티팩트 및 파이프라인 통합**
   - 48단계에서 검증한 아티팩트 저장소와 연결하여 이미지/패키지를 가져오도록 설정한다.
   - 파이프라인 스테이지 정의를 업로드하고 환경 변수/시크릿을 주입한다.
5. **배포 타겟 등록**
   - EC2 인스턴스, Auto Scaling 그룹, ECS 서비스, EKS 네임스페이스 등 배포 대상 리소스를 등록한다.
   - 헬스체크/롤백 설정을 32~33단계 전략과 일치시키고 태그 기반 타겟팅을 구성한다.
6. **검증 및 모니터링 연계**
   - 샘플 배포(드라이런 또는 소규모 트래픽)를 실행해 성공 여부를 확인한다.
   - 26단계 모니터링과 연동하여 배포 이벤트를 기록하고 알림을 설정한다.

## 의사결정 포인트
- CD 도구를 멀티 환경(dev/test/prod)에 공유할지 환경별로 분리할지 여부.
- 배포 승인/검토 단계(34단계)를 CD 도구 내에서 처리할지 외부 워크플로우로 처리할지 결정.
- 배포 실패 시 자동 롤백을 기본값으로 설정할지 여부.

## 체크리스트
- [ ] CD 도구가 설치/구성되어 dev 환경과 통신한다.
- [ ] IAM/시크릿/아티팩트 연동이 성공적으로 완료되었다.
- [ ] 배포 대상 리소스가 등록되고 헬스체크/롤백 설정이 적용되었다.
- [ ] 샘플 배포 또는 드라이런이 성공적으로 수행되었다.

## 산출물 및 보관 위치
- CD 도구 설치 IaC/스크립트 (`infra/cd/`, `scripts/cd-bootstrap/`).
- 배포 파이프라인 정의 (`cd/pipelines/*.yaml`).
- 검증 보고서 및 배포 로그 (`docs/reports/cd-bootstrap-validation.md`).
