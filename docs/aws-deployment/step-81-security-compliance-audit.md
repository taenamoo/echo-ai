# 81단계. 보안 정책 준수 감사 실시

## 목적
- AWS 계정과 워크로드가 7단계에서 정의한 보안 및 규제 요구사항을 충족하는지 검증한다.
- 70단계 보안 점검 결과와 비교하여 잔여 위험을 파악하고, 운영 이전에 시정 조치를 완료한다.

## 선행조건
- [70단계 보안 점검](docs/aws-deployment/step-70-security-review.md) 결과 보고서 및 개선 항목 가용.
- [76단계 감사 추적 검토](docs/aws-deployment/step-76-audit-trail-review.md)에서 수집된 로그와 보존 정책이 활성화되어 있어야 한다.
- [75단계 보안 사고 대응 절차](docs/aws-deployment/step-75-security-incident-response.md)에서 정의한 대응 체계가 문서화되어 있어야 한다.

## 필요한 입력 자료
- 보안 정책/규제 요구사항 목록, 체크리스트, 표준 운영 절차(SOP).
- AWS Security Hub, IAM Access Analyzer, Config Rules, Trusted Advisor 등 감사 리포트.
- 최근 3개월 이상 CloudTrail, Config, VPC Flow Logs 등의 감시 로그.

## 상세 절차
1. **감사 범위 및 기준 확정**
   - 계정, 리전, 서비스 범위를 정의하고 필수 규제/정책 통제 항목을 확정한다.
   - 7단계 보안 요구사항 매트릭스와 31단계 품질 기준을 참고하여 평가 항목을 정렬한다.
2. **감사 도구 실행 및 증적 수집**
   - Security Hub/Config Rules를 실행하여 위반 리소스를 식별하고 증적(스크린샷, 리포트)을 보관한다.
   - 수동 검증이 필요한 항목은 CLI/콘솔을 통해 추가 증적을 수집한다.
3. **위반 항목 분석 및 우선순위 지정**
   - 위험도와 영향도를 평가하여 시정 순서를 결정하고, 운영 영향과 일정(65단계 배포 일정)을 검토한다.
   - IAM/네트워크 관련 항목은 12·17단계 설계와 비교하여 차이점을 문서화한다.
4. **시정 조치 실행 및 검증**
   - IaC(4단계 결정) 또는 수동 작업으로 정책을 수정하고 재감사를 수행해 준수 여부를 확인한다.
   - 변경 사항이 76단계 감사 로그에 기록되는지 검증한다.
5. **감사 보고서 작성 및 공유**
   - 위반 항목, 시정 상태, 잔여 위험, 후속 계획을 보고서로 정리한다.
   - 74단계 운영 핸드오프, 92단계 보안 교육과 공유하여 운영 인수 전 인지시키는다.

## 의사결정 포인트
- 감사 주기를 분기/반기 중 어느 주기로 설정할지 여부.
- 내부 감사팀, 외부 감사 기관 등 참여 범위 결정.
- 시정 조치에 IaC 배포를 사용할지 수동 수정 후 추후 코드화할지 선택.

## 체크리스트
- [ ] 감사 범위와 기준이 문서화되고 승인되었다.
- [ ] 감사 도구 결과와 수동 증적이 수집되어 위반 항목이 식별되었다.
- [ ] 시정 조치가 완료되어 재검증 결과 준수 상태임을 확인했다.
- [ ] 최종 감사 보고서가 이해관계자(보안/운영/경영)에 공유되었다.
