# 40단계. 개발 환경용 네트워크 구축

## 목적
- 설계된 네트워크 아키텍처에 따라 개발(dev) 환경용 VPC/서브넷/보안 구성을 실제로 구축한다.
- 향후 스테이징/운영 환경 확장을 위한 표준 템플릿과 검증 절차를 마련한다.

## 선행조건
- 14~19단계 네트워크 설계 및 테스트 계획.
- 16단계 VPC/서브넷 구현 계획과 17단계 보안 정책.
- 32단계 배포 전략, 33단계 롤백 정책, 39단계 DR 전략.

## 필요한 입력 자료
- 개발 환경 리소스 요구사항(컴퓨팅, 데이터베이스, 툴링 등).
- 네트워크 주소 계획(CIDR, 서브넷 분할), 라우팅/피어링 계획.
- 접근 제어 정책(IAM, 보안 그룹, NACL), 로깅/모니터링 요구.

## 상세 절차
1. **구축 범위 확정**
   - 개발 환경에서 필요한 VPC 수, 서브넷 유형(공용/사설), AZ 구성을 확정한다.
   - 32단계 배포 전략에서 요구하는 트래픽 경로를 확인한다.
2. **IaC 또는 수동 절차 실행**
   - 4단계 IaC 결정에 따라 Terraform/CDK/CloudFormation 템플릿을 실행한다.
   - 수동일 경우 표준 운영 절차(SOP)를 따라 생성하고 실행 로그를 보관한다.
3. **네트워크 보안 구성 적용**
   - 17단계에서 설계한 보안 그룹/NACL/방화벽 규칙을 구현한다.
   - VPC 엔드포인트, NAT 게이트웨이, 프록시 구성 등 보안 경계 요소를 배치한다.
4. **연결성 및 회복력 검증**
   - 19단계 테스트 계획에 따라 VPC 피어링, VPN, Direct Connect 연결성을 검증한다.
   - 39단계 DR 요구에 따라 교차 리전/계정 연결 테스트를 수행한다.
5. **모니터링 및 로깅 설정**
   - VPC Flow Logs, CloudTrail, GuardDuty 등을 활성화하고 26단계 모니터링 플랫폼과 연계한다.
   - 36단계 비용 모니터링을 위해 NAT/데이터 전송 비용 추적을 설정한다.
6. **문서화 및 운영 인수인계**
   - 네트워크 다이어그램, 서브넷 테이블, 라우팅 테이블을 최신화한다.
   - 운영팀 인수인계 문서와 접근 절차를 작성한다.

## 의사결정 포인트
- 개발 환경과 공유 서비스 계정의 분리 여부.
- 인터넷 게이트웨이/NAT 게이트웨이 구성 방식과 비용 최적화 전략.

## 체크리스트
- [ ] VPC/서브넷/라우팅 리소스가 계획대로 생성되었다.
- [ ] 보안 그룹/NACL 등 보안 구성이 적용되었다.
- [ ] 연결성 테스트와 DR 연계 검증이 완료되었다.
- [ ] 모니터링/로깅이 활성화되고 문서화되었다.
