# 20단계. 컴퓨팅 서비스 선택

## 목적
- 애플리케이션 요구사항에 맞는 AWS 컴퓨팅 서비스(EC2, ECS, EKS, Lambda 등)를 평가하고 선택한다.
- 성능, 확장성, 운영 비용, 팀 역량을 고려한 배포 대상 환경을 확정한다.

## 선행조건
- 0단계 요구사항 명세와 2단계 애플리케이션 아키텍처.
- 3단계 버전 전략, 6단계 비용 계획, 7단계 보안 요구사항.
- 14~19단계 네트워크 및 인프라 설계 결과.

## 필요한 입력 자료
- 애플리케이션 런타임 특성(언어, 프레임워크, 상태 유지 여부).
- 성능 지표(응답 시간, 처리량), 확장 요구, SLA.
- 팀의 운영 역량(컨테이너, 쿠버네티스, 서버리스 경험)과 도구 체계.

## 상세 절차
1. **요구사항 분석**
   - 기능/비기능 요구사항을 정리하고, 컴퓨팅 서비스별 적합성을 매트릭스로 비교한다.
   - 배포 모델(컨테이너, VM, 서버리스) 및 상태 관리 요구사항을 식별한다.
2. **후보 서비스 평가**
   - EC2 Auto Scaling, ECS(Fargate/EC2), EKS, Lambda, App Runner 등 후보를 선정한다.
   - 비용, 운영 복잡도, 보안 제어 수준, 멀티 리전 지원 여부를 비교한다.
3. **PoC 또는 벤치마크 수행**
   - 샘플 워크로드를 배포하여 성능, 확장성, 관측성 요구를 검증한다.
   - IaC/CI 파이프라인과 통합 테스트를 수행하여 운영 가능성을 평가한다.
4. **운영 및 배포 전략 수립**
   - 선택한 서비스에 맞는 배포 전략(32단계) 및 롤백 전략(33단계)을 미리 구상한다.
   - 모니터링, 로깅, 패치 관리 방안을 정의한다.
5. **비용 및 라이선스 검토**
   - TCO(총 소유 비용) 분석, Savings Plans/RI 적용 가능성, 라이선스 영향(Windows, BYOL 등)을 검토한다.
   - 6단계 예산과 비교하여 승인 절차를 진행한다.
6. **보안 및 규제 검토**
   - 선택한 서비스가 보안 요구를 충족하는지 확인하고, 추가 통제 수단을 정의한다.
   - 데이터 분류에 따른 암호화, 네트워크 격리, 감사 로깅을 검토한다.
7. **결정 문서화 및 승인**
   - 최종 선택 이유, 비교 결과, PoC 데이터, 위험 요소를 문서화한다.
   - 이해관계자 승인 후 후속 설계/구현 일정에 반영한다.

## 의사결정 포인트
- 컨테이너 오케스트레이션(ECS/EKS) vs. 서버리스(Lambda/App Runner) 선택.
- 관리형 서비스 사용 범위와 자체 운영 필요성.

## 체크리스트
- [ ] 컴퓨팅 서비스 비교 평가표와 PoC 결과가 작성되었다.
- [ ] 비용/보안/운영 검토가 완료되었다.
- [ ] 배포/운영 전략 초안이 수립되었다.
- [ ] 최종 결정이 승인되었고 문서화되었다.
