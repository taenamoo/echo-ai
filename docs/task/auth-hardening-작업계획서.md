## 작업 계획서: 인증/보안 강화 및 라우트 정리
우선순위: 5

### 1. 목표
- 인증 라우트를 `/api/auth/*`로 일원화하고, 중복/구 라우트를 제거한다.
- 토큰 만료·검증을 강화하고, 기본 비밀번호 정책을 도입한다.

-----

### 2. 사전 조건
- 현행 라우트: `/api/auth/login`, `/api/auth/signup`, (구) `/api/signup`
- 토큰 유틸: `lib/auth/token` 사용

-----

### 3. 작업 단계 (Step-by-Step)
1) 라우트 정리
  - 파일 제거: `src/app/api/signup/route.ts`
  - import/사용처 확인 및 삭제

2) 토큰 만료 강화
  - `generateAccessToken(userId, email, { expiresIn: '1h' })` 적용
  - 미들웨어/유틸에서 만료 에러 메시지 구분

3) 비밀번호 정책
  - 규칙: 최소 8자, 숫자/문자 조합, 공백 금지
  - 회원가입 시 검증, 실패 시 400 반환

4) 에러 메시지 정비
  - 인증 실패/권한 없음/입력 오류 케이스별 표준화

5) (선택) 이메일 검증/재설정 설계
  - 후속 이슈로 분리(토큰 발급/링크 검증 흐름 정의)

-----

### 4. 수용 기준
- `/api/signup` 제거, 모든 인증 트래픽이 `/api/auth/*`로 동작
- 만료 토큰 거절 동작 검증, 비밀번호 정책 적용
